Writing Algorithms

An algorithm is a step-by-step solution to a problem — the plan you write before any C syntax exists, so the code only has to express an idea you already understand clearly.

What is an Algorithm?

Definition: An algorithm is a finite sequence of well-defined steps used to solve a problem or perform a task.

In simple words: Algorithm = Step-by-step solution to a problem.

Real-Life Example

Problem: Brush Your Teeth

Algorithm 
Start
Pick up the toothbrush.
Apply toothpaste.
Brush your teeth.
Rinse your mouth.
Stop.

Notice that every step is written in order.

Why Do We Need an Algorithm?

Imagine writing a C program without knowing the solution first. You may:

  • Miss important steps.
  • Write incorrect code.
  • Spend more time debugging.

An algorithm helps you:

  • Plan the solution.
  • Reduce errors.
  • Write code faster.
  • Make programs easier to understand.

Characteristics of a Good Algorithm

A good algorithm should have these 5 properties:

1. Input

It should accept zero or more inputs.

Example 
Number = 10

2. Output

It should produce at least one output.

Example 
Square = 100

3. Definiteness

Every step should be clear and unambiguous.

❌ Bad: Calculate something.

✅ Good: Multiply Length × Width.

4. Finiteness

The algorithm must end after a finite number of steps.

❌ Wrong: Repeat forever.

✅ Correct: Repeat 10 times.

5. Effectiveness

Each step should be simple and executable.

Example 
Read a number.
Add 5.
Display the result.

General Structure of an Algorithm

Start
Input
Process
Output
Stop

This is also called the IPO (Input–Process–Output) Model.

Example 1: Add Two Numbers

Problem: Find the sum of two numbers.

Algorithm 
Step 1: Start
Step 2: Read A
Step 3: Read B
Step 4: Sum = A + B
Step 5: Display Sum
Step 6: Stop

Example

Input:

Input 
A = 10
B = 20

Output:

Output 
Sum = 30

Problem: Check whether a number is even or odd.

Algorithm

Step 2: Read Number

Step 4: If remainder is 0
            Display "Even"

            Display "Odd"

            Display A

Advantages of Algorithms

  • Easy to understand.
  • Independent of any programming language.
  • Helps find mistakes before coding.
  • Makes debugging easier.
  • Saves development time.

Disadvantages

  • Writing algorithms for very large systems can be time-consuming.
  • Complex problems may require lengthy algorithms.

Algorithm vs Program
