Operating Systems

C and Operating Systems are closely connected — most operating systems (or major parts of them) are written in C. Linux Kernel, UNIX, parts of Windows, and the Android Kernel are all examples.

What is an Operating System (OS)?

Definition: An Operating System (OS) is system software that acts as an interface between the user and the computer hardware.

In simple words: Operating System = Manager of the Computer. It controls all hardware and software resources.

Real-Life Example

Imagine a restaurant. Customer → User. Chef → Hardware. Waiter → Operating System.

The customer doesn't go directly to the chef. Instead:

The waiter takes the order, gives it to the chef, and brings back the food. Similarly:

The OS communicates between you and the hardware.

Why Do We Need an Operating System?

Without an Operating System, you cannot:

  • Open files.
  • Run programs.
  • Use the keyboard or mouse.
  • Connect to the internet.
  • Print documents.

The OS manages all these tasks.

Functions of an Operating System

An OS performs many important functions.

1. Process Management

A process is a program that is currently running. Example: you open Chrome, VS Code, and Spotify — each one is a separate process.

The OS creates processes, runs them, stops them, and schedules CPU time.

VS Code
Music Player

The CPU switches between them very quickly.

2. Memory Management

RAM is limited. The OS decides which program gets memory, how much memory is allocated, and when memory is released.

3. File Management

The OS manages files, folders, and directories. Examples: Documents, Pictures, Videos, Downloads. Without the OS, organizing files would be extremely difficult.

4. Device Management

The OS controls hardware devices like the keyboard, mouse, printer, scanner, monitor, and webcam.

When you press a key:

Application

5. Security

The OS protects your computer using passwords, user accounts, permissions, encryption, and firewalls (often with additional software).

Example: a Windows login screen asks for a Username and Password. Only authorized users can access the system.

6. User Interface

The OS provides an interface for interacting with the computer. Two common types:

GUI (Graphical User Interface) — Examples: Windows, macOS, Ubuntu Desktop. Uses icons, windows, mouse, buttons.

CLI (Command Line Interface) — Examples: dir, cd, ls, mkdir. Used in the Linux Terminal, Windows Command Prompt, and PowerShell.

7. Resource Management

The OS manages the CPU, RAM, Hard Disk/SSD, printers, USB devices, and network resources. It ensures multiple programs can share these resources efficiently.

Types of Operating Systems

1. Batch Operating System

Oldest type. Jobs are collected into batches and executed one after another.

Job 1
Job 2
Job 3

No user interaction during execution.

2. Time-Sharing Operating System

Allows multiple users or programs to share the CPU. The CPU rapidly switches between tasks.

Chrome
VS Code
Spotify
Back to Chrome

This creates the illusion that everything runs simultaneously. Examples: Linux, UNIX.

3. Multitasking Operating System

Runs multiple applications at the same time. Example: YouTube, MS Word, Calculator, VS Code. Examples: Windows, macOS, Linux.

4. Multiuser Operating System

Allows multiple users to use the same system. Example: a university server with many student accounts.

5. Real-Time Operating System (RTOS)

Designed for systems where timing is critical. Examples: airplane control systems, medical equipment, industrial robots, car braking systems (ABS).

Popular RTOS examples: FreeRTOS, VxWorks, QNX.

6. Distributed Operating System

Controls multiple computers and makes them appear as one system. Used in supercomputers and large clusters.

Why is C Used to Develop Operating Systems?

This is a very common interview question. Reasons:

  • Fast execution.
  • Direct memory access using pointers.
  • Efficient hardware interaction.
  • Small runtime overhead.
  • Portable across different hardware.

Interesting Facts 💡

  • The Linux Kernel contains millions of lines of C code.
  • Android is built on the Linux kernel.
  • The first version of UNIX was rewritten in C, making it much easier to move to different hardware.
  • C remains one of the most important languages for systems programming.

Quick Revision

  • An Operating System manages computer hardware and software.
  • It acts as an interface between the user and hardware.
  • Main functions: Process Management, Memory Management, File Management, Device Management, Security, User Interface.
  • The Kernel is the core of the OS.
  • The Shell lets users communicate with the kernel.
  • C is widely used for operating system development because of its speed and low-level capabilities.

Interview Questions

Beginner

An Operating System is system software that acts as an interface between the user and the computer hardware, managing all hardware and software resources.

Without an OS, you couldn't open files, run programs, use the keyboard or mouse, connect to the internet, or print documents — the OS manages all these tasks.

Process management, memory management, file management, device management, security, user interface, and resource management.

A GUI (Graphical User Interface) uses icons, windows, and a mouse to interact with the computer (e.g. Windows, macOS). A CLI (Command Line Interface) uses typed text commands like dir, cd, ls, and mkdir (e.g. Linux Terminal, PowerShell).

The kernel is the core (heart) of the Operating System — it directly communicates with the hardware and handles CPU scheduling, memory management, device management, and process management.

Intermediate

The kernel is the core of the OS that directly communicates with hardware and manages CPU, memory, and devices. The shell is the interface between the user and the kernel — it takes user commands (like in Bash or PowerShell) and passes them to the kernel.

Batch OS runs jobs one after another with no user interaction. Time-Sharing OS (e.g. Linux, UNIX) shares CPU among multiple users/programs. Multitasking OS (e.g. Windows, macOS) runs multiple applications at once. Multiuser OS allows many users on one system (e.g. a university server). Real-Time OS (e.g. FreeRTOS, VxWorks, QNX) is used where timing is critical, like airplane control systems. Distributed OS controls multiple computers appearing as one system, used in supercomputers and clusters.

Because C offers fast execution, direct memory access using pointers, efficient hardware interaction, small runtime overhead, and portability across different hardware — exactly what's needed to build an OS.

Process management involves creating, running, stopping, and scheduling CPU time for processes (running programs) like Chrome, VS Code, and Spotify. Memory management involves deciding which program gets memory, how much is allocated, and when it is released, since RAM is limited.

A file system is the method the OS uses to organize and store files. Common examples: NTFS, FAT32, and exFAT on Windows; ext4, XFS, and Btrfs on Linux; and APFS on macOS.
