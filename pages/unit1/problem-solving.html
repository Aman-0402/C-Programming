Problem Solving with Sequential Logic Structure

Before you write a single line of C, you need to understand what problem solving and logic actually mean — and how the simplest control structure of all, sequential logic, executes your instructions one after another.

What is Problem Solving?

Problem Solving is the process of understanding a problem, finding a solution, and then implementing it in a programming language.

Problem: You are hungry.

Solution Steps:

  1. Go to the kitchen.
  2. Take bread and butter.
  3. Apply butter on bread.
  4. Eat the bread.

You don't start eating before making the bread! The steps happen in the correct order. Programming works the same way.

What is Logic?

Logic is the sequence of decisions or steps used to solve a problem. A programmer thinks logically before writing code.

Problem: Find the area of a rectangle.

Logic:

  1. Read length.
  2. Read width.
  3. Multiply them.
  4. Display the result.

Simple formula:

Formula 
Area = Length × Width

What is a Sequential Logic Structure?

A Sequential Logic Structure means that statements execute one after another in the order they are written, without skipping or repeating. It is the simplest control structure in programming.

Start
Step 1
Step 2
Step 3
Stop

Every step is executed exactly once.

Example 1: Making Tea

Imagine making tea.

Start
Boil water
Add tea leaves
Add sugar
Add milk
Serve tea
Stop

If you add milk before boiling water, the process won't work correctly. This is why order matters.

Example 2: Add Two Numbers

Suppose:

Given 
A = 10
B = 20

Steps:

Start
Read A
Read B
Sum = A + B
Print Sum
Stop

Output:

Output 
30

C Program Example

add.c 
#include <stdio.h>

int main()
{
    int a = 10;
    int b = 20;
    int sum;

    sum = a + b;

    printf("Sum = %d", sum);

    return 0;
}

Execution Order

  1. Declare variables
  2. Store values
  3. Calculate sum
  4. Print result
  5. End program

Every line executes from top to bottom.

Another Example: Student Percentage

Suppose:

Given 
Marks = 450
Total = 500

Formula:

Formula 
Percentage = (Marks × 100) / Total

Steps:

Start
Read Marks
Read Total
Calculate Percentage
Display Percentage
Stop

Output:

Output 
90%

Characteristics of Sequential Logic

  • Executes statements one by one.
  • No conditions (if, else).
  • No loops (for, while).
  • Easy to understand.
  • Used in almost every program.

Advantages

  • Very simple.
  • Easy to debug.
  • Easy to read.
  • Best for beginner programs.
  • Suitable for straightforward calculations.

Limitations

  • Cannot make decisions.
  • Cannot repeat tasks automatically.
  • Not suitable for complex programs.

Example: A login system needs to check whether the password is correct. Sequential logic alone cannot do this — you need decision-making (if-else).

Where is Sequential Logic Used?

  • Calculator programs
  • Area and perimeter calculations
  • Unit conversions
  • Temperature conversion
  • Simple billing systems
  • Basic mathematical operations

Each step happens in sequence.

Interesting Facts 💡

  • Every program starts with sequential execution.
  • Even advanced software like games or web browsers execute instructions sequentially unless a condition or loop changes the flow.
  • There are three basic control structures in programming: Sequential, Selection (Decision Making), and Iteration (Loops).

Quick Revision

  • Problem solving means finding a logical solution before coding.
  • Logic is the sequence of steps used to solve a problem.
  • Sequential logic executes instructions top to bottom.
  • No decisions or loops are involved.
  • It is the first and simplest control structure.

MCQs

Interview Questions

Beginner

Problem solving is the process of understanding a problem, finding a solution, and then implementing it in a programming language.

Logic is the sequence of decisions or steps used to solve a problem — a programmer thinks logically before writing code.

Sequential logic means statements execute one after another in the order they are written, without skipping or repeating — it's the simplest control structure in programming.

Because order determines correctness — a step run before its required input is ready produces wrong or undefined results. Every program relies on this default before any decisions or loops are added.

ATM cash withdrawal (insert card → enter PIN → select amount → receive cash → take card) and making tea (boil water → add tea leaves → add sugar → add milk → serve).

Intermediate

Sequential runs statements one after another with no branching or repetition. Selection (decision making, e.g. if-else) chooses between different paths based on a condition. Iteration (loops, e.g. for, while) repeats a set of statements multiple times.

A login system must check whether the entered password is correct and respond differently depending on the result — that requires decision-making (if-else), which sequential logic alone doesn't provide.

Input is data the program reads (e.g. two numbers), Process is what it does with that data (e.g. adding them), and Output is the result shown to the user (e.g. printing the sum). Sequential logic naturally follows this order: read inputs, process them, then output the result.

In calculator programs, area/perimeter calculations, unit and temperature conversions, simple billing systems, and any straightforward step-by-step calculation without branching or repetition.
