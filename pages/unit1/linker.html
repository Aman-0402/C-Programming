Linker

The stage that combines your compiled object code with any library code it needs, producing one complete, runnable executable.

1. What Is It?

The linker combines one or more object code files with any needed library code — such as the compiled implementation of printf — into a single, complete executable. It is the stage that takes the separate pieces produced earlier in the pipeline and assembles them into one whole program.

2. Why Do We Need to Know This?

This explains why a C program that only calls printf still ends up as a complete, standalone program. Your own object code doesn't contain printf's actual instructions — the linker pulls in printf's real implementation from the C standard library and merges it with your code, so the final executable has everything it needs to run on its own.

3. Real-Life Example

Like a publisher combining a manuscript with pre-printed cover art and binding materials into a finished, sellable book. The manuscript alone isn't a book you can sell — each piece exists separately until the publisher's assembly process links them together into one finished, complete product.

5. Common Mistakes

  • Assuming your own object code alone is a complete program — it usually isn't, since it still needs library code linked in to run.
  • Mistaking "undefined reference" linker errors for compiler errors — they actually happen at the separate linking stage, after compilation has already succeeded.

6. Debugging Tips

If you see an error mentioning "undefined reference," that's a linker error — a missing library or a missing function definition — not a compilation or syntax error. The code compiled fine; the linker just couldn't find something it needed to link against.

7. Best Practices

  • Know that "undefined reference" errors point to linking, not syntax.
  • When you see one, look for missing library flags or missing function definitions, not typos in your syntax.

12. FAQs

Q: Do I need to link libraries manually?
A: No — gcc handles the standard library automatically for simple programs.

Q: Is linking part of compilation?
A: Yes — it's the final internal stage of the overall compilation process.

13. Key Takeaways

  • The linker combines your object code with any needed library code into one complete executable.
  • Even a simple printf program relies on the linker pulling in the standard library's actual implementation.
  • "Undefined reference" errors happen at the linking stage, not during compilation.
