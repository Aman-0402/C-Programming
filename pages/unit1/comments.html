Single-Line and Multi-Line Comments

Comments may seem like a small topic, but they are extremely important in real-world programming. Professional developers write comments to explain code, making it easier for themselves and others to understand programs later.

What is a Comment?

Definition: A comment is text written inside a program that is ignored by the compiler. Comments are written only for humans, not for the computer.

Simply put: Comment = Explanation for the programmer.

Real-Life Example

Imagine reading a book. Sometimes the author writes notes like:

Note 
Note:
This formula will be used in the next chapter.

This note is for the reader, not part of the actual content. Similarly:

Example 
// Calculate the total marks

is for the programmer, not the computer.

Why Do We Use Comments?

Comments help to:

  • Explain the purpose of the code.
  • Make programs easier to understand.
  • Help during debugging.
  • Make maintenance easier.
  • Improve teamwork.
  • Temporarily disable code while testing.

Types of Comments in C

C supports two types of comments:

  1. Single-Line Comments (//)
  2. Multi-Line Comments (/* ... */)

Single-Line Comment

Syntax

Syntax 
// This is a comment

Everything after // on the same line is ignored by the compiler.

Example 1

hello.c 
#include <stdio.h>

int main()
{
    // Print Hello World
    printf("Hello World");

    return 0;
}

Output: Hello World

The compiler ignores // Print Hello World.

Example 2

Example 
int age = 20; // Store age

The program stores 20. The text "Store age" is ignored.

Multiple Single-Line Comments

Example 
// Read input

// Calculate sum

// Display result

Each line starts with //.

Multi-Line Comment

Syntax

Syntax 
/*
This is a
multi-line
comment
*/

Everything between /* and */ is ignored.

Example

hello.c 
#include <stdio.h>

/*
Program Name : Hello World
Author       : Aman
Version      : 1.0
*/

int main()
{
    printf("Hello");

    return 0;
}

Output: Hello

The entire block comment is ignored.

Another Example

Example 
/*
Calculate Area

Formula

Area = Length × Width
*/

Useful for describing algorithms, formulas, or program details.

Difference Between Single-Line and Multi-Line Comments

Single-Line CommentMulti-Line Comment
Starts with //Starts with /*
Ends at the end of the lineEnds with */
Used for short explanationsUsed for long descriptions
One line onlyMultiple lines

Can We Write Comments Anywhere?

Yes.

Example 
// Header comment

#include <stdio.h>

int main()
{
    // Variable declaration
    int a = 10;

    // Printing value
    printf("%d", a);

    return 0;
}

Comments can appear before, after, or between statements.

Using Comments for Documentation

Professional programs often begin with:

Example 
/*
Program Name : Student Management System
Author       : Aman
Date         : 23 July 2026
Version      : 1.0
Description  : Stores student records.
*/

This makes the code easier to understand.

Using Comments to Disable Code

Sometimes you don't want to delete code. Instead, comment it out.

Example 
printf("Hello");

/*
printf("Welcome");
printf("Learning C");
*/

printf("Bye");

Output:

Output 
Hello
Bye

The commented code is ignored.

Common Beginner Mistakes

Mistake 1: Forgetting to Close a Multi-Line Comment

Wrong:

broken.c 
/*
This comment never ends

printf("Hello");

The compiler treats the rest of the file as part of the comment, leading to an error.

Correct:

fixed.c 
/*
This comment ends properly.
*/

printf("Hello");

Mistake 2: Nested Multi-Line Comments

Wrong:

broken.c 
/*
Outer comment

/* Inner comment */

End
*/

Standard C does not support nested block comments.

Correct approach:

fixed.c 
/*
Outer comment
Inner comment
End
*/

or use // inside a block comment if appropriate.

Mistake 3: Writing Too Many Comments

Bad:

Example 
// Declare integer
int a;

// Assign value
a = 10;

// Print value
printf("%d", a);

The code is already obvious.

Better:

Example 
// Store the student's total marks
int totalMarks = 450;

Good comments explain why, not just what.

Good vs Bad Comments

❌ Bad:

Example 
// Add 1 to i
i = i + 1;

The code already tells us that.

✅ Good:

Example 
// Skip the first record because it contains column headings.
i = i + 1;

This explains the reason.

Best Practices

  • ✔ Write meaningful comments.
  • ✔ Keep comments updated.
  • ✔ Explain complex logic.
  • ✔ Don't comment every obvious line.
  • ✔ Remove unnecessary comments before final submission if they add no value.

Interesting Facts 💡

  • Comments do not affect program execution because the compiler ignores them.
  • Many IDEs can automatically generate documentation comments.
  • Well-commented code is easier to maintain, especially in team projects.

Quick Revision

  • Comments are ignored by the compiler.
  • Comments improve readability and maintainability.
  • // is used for single-line comments.
  • /* ... */ is used for multi-line comments.
  • Avoid unnecessary or outdated comments.
  • Standard C does not support nested multi-line comments.
