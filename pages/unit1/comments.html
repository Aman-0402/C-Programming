Comments

Text left behind for human readers, not for the compiler — the opening topic of this chapter, before the next two pages dig into C's two comment styles.

1. What Is It?

A comment is text in source code that the compiler completely ignores — it exists purely for human readers, not for the computer. C supports two comment styles, single-line and multi-line, each covered in detail on the next two pages.

2. Why Do We Need to Know This?

Comments let you explain why code does something, not just what it does — invaluable for your future self and anyone else reading your code later.

3. Real-Life Example

Like a sticky note left in a recipe book explaining why you substitute an ingredient — the recipe still works without it, but the note helps whoever reads it next understand your reasoning.

4. Visual Explanation

Here is add.c again, this time with an explanatory comment added:

add.c 
#include <stdio.h>

int main() {
    // Print the result of adding 5 and 3
    printf("The sum of 5 and 3 is 8\n");
    return 0;
}

The comment line is completely ignored by the compiler — removing it wouldn't change the program's behavior at all. It exists solely to help a human reader understand the line below it.

5. Common Mistakes

  • Assuming comments affect how a program runs — they never do; they're purely for humans.
  • Over-commenting obvious code, such as commenting return 0; with "return zero".
  • Under-commenting genuinely tricky or non-obvious logic, where a comment would actually help.

6. Debugging Tips

  • A classic technique called "commenting out" temporarily disables a line or block of code by turning it into a comment, letting you test a program without that code running, without deleting it.
  • The next two pages explain exactly how to apply this technique using each comment style.

7. Best Practices

  • Comment the why, not the what, when the code itself is already clear.
  • Keep comments up to date — an outdated comment describing old behavior is worse than no comment at all.

12. FAQs

Q: Do comments make a program run slower?
A: No — they're stripped out before execution, so they have zero runtime cost.

Q: Are comments required in C?
A: No, entirely optional — but strongly recommended for readability.

13. Key Takeaways

  • A comment is text the compiler completely ignores — it exists purely for human readers.
  • Follow the why-not-what principle: comment the reasoning, not what the code already makes obvious.
  • "Commenting out" is a classic debugging technique for temporarily disabling code without deleting it.
