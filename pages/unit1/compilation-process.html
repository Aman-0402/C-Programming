Compilation (Source Code → Object Code → Executable Code)

Many beginners think that when they press Run, the program executes immediately. It doesn't. Before a C program runs, it goes through multiple stages.

Big Picture

When you write a C program, the computer cannot understand C language directly. It understands only Machine Language (0s and 1s). So your program goes through this journey:

Source Code (.c)
Preprocessor
Compiler
Assembler
Object Code (.o/.obj)
Linker
Executable File (.exe on Windows)
Loader
Program Runs

Understanding these stages will help you debug compilation errors, understand interview questions, and learn how a compiler works internally.

Real-Life Analogy

Imagine writing a book.

You write a manuscript.
Editor checks grammar.
Printing press converts it into a printed book.
Book is delivered to readers.

Similarly,

Write Program
Compiler Checks
Machine Code Created
Executable Generated
Program Runs

Step 1: Source Code

This is the code written by the programmer. Extension: .c

hello.c 
#include <stdio.h>

int main()
{
    printf("Hello");

    return 0;
}

This is called the Source Code.

What is Source Code?

Definition: Source Code is the human-readable program written in a programming language such as C.

Step 2: Preprocessing

Before compilation begins, the Preprocessor processes the source code. It handles:

  • #include
  • #define
  • Conditional compilation directives like #ifdef, #ifndef, #if

Example: #include <stdio.h>

The preprocessor replaces this with the contents of the stdio.h header file. Think of it as:

#include <stdio.h>
Entire stdio.h file inserted here

It also removes comments.

Before 
// This is a comment

printf("Hello");

After preprocessing:

After 
printf("Hello");

The comment is removed.

Step 3: Compilation

Now the Compiler checks the program. It checks for:

  • Syntax errors
  • Type checking
  • Variable declarations
  • Function definitions

Correct: int a = 10;

Wrong: int a = ;

Compiler error: Syntax Error

If everything is correct, the compiler converts the code into Assembly Language.

Step 4: Assembling

The Assembler converts the Assembly Language into Machine Code. Machine Code contains only binary instructions.

Example 
1011010010101010

The output is called Object Code. Extensions: Windows .obj, Linux/macOS .o.

What is Object Code?

Object Code is machine code generated from one source file, but it is not yet a complete runnable program. It may still contain unresolved references to library functions like printf().

Step 5: Linking

Now the Linker takes your Object Code and Required Library Files, and combines them.

Your program contains printf("Hello");. But where is printf()? It exists inside the Standard C Library. The linker connects your code with that library.

After successful linking: Executable File Created. Examples: Windows hello.exe, Linux hello.

What is an Executable File?

An Executable File is a complete machine-language program that the operating system can run. This is what you execute when you double-click a program or type its name in the terminal.

Step 6: Loading

The Loader is part of the operating system. It:

  • Loads the executable into RAM (memory).
  • Allocates memory for the program.
  • Prepares the program to run.
  • Transfers control to the main() function.

Step 7: Execution

Finally, the CPU starts executing the instructions. The program begins at int main().

Example 
printf("Hello");

Output: Hello

What Happens When You Click "Run"?

When you click Run in an IDE (such as Code::Blocks, Dev-C++, Visual Studio Code with GCC, or CLion), it usually performs these steps automatically:

  1. Save the source file.
  2. Preprocess the code.
  3. Compile the code.
  4. Assemble it.
  5. Link it.
  6. Load it into memory.
  7. Execute the program.

You don't see these steps because the IDE performs them behind the scenes.

Compilation Errors vs Runtime Errors

Compilation Error — Occurs before the program runs.

broken.c 
#include <stdio.h>

int main()
{
    printf("Hello")

    return 0;
}

Missing semicolon: Compilation Error. The executable is not created.

Runtime Error — Occurs after successful compilation, while the program is running.

Example 
int a = 10;
int b = 0;

printf("%d", a / b);

Division by zero can cause a runtime error or undefined behavior depending on the system.

Interesting Facts 💡

  • Large software projects can contain thousands of object files before linking.
  • The Linux kernel consists of millions of lines of C code and is built by compiling many source files into object files and then linking them together.
  • GCC can stop after different stages (preprocessing, compilation, assembling) using command-line options, which is useful for debugging and learning.

Quick Revision

  • Source Code → Human-readable C program (.c).
  • Preprocessor → Handles #include, #define, removes comments.
  • Compiler → Checks syntax and converts C to Assembly.
  • Assembler → Converts Assembly to Object Code (.o/.obj).
  • Linker → Combines object files with libraries to create an executable.
  • Loader → Loads the executable into memory.
  • Execution → CPU runs the program starting from main().

MCQs

Interview Questions

Beginner

Source code is the human-readable program written by the programmer in a language such as C, usually saved with a .c extension.

Object code is machine code generated by the assembler from one source file, but it is not yet a complete runnable program — it may still contain unresolved references to library functions.

An executable file is a complete machine-language program that the operating system can run, produced after the linker combines object code with the required libraries.

The compiler checks the preprocessed source code for syntax errors, type checking, variable declarations, and function definitions, then converts it into Assembly Language if everything is correct.

Compilation converts source code into machine code (done by the compiler, assembler, and linker) and happens before running, detecting syntax errors. Execution runs the machine code (done by the CPU after loading) and happens after compilation, potentially revealing runtime or logical errors.

Intermediate

Source code (.c) is first handled by the preprocessor (expanding #include/#define, removing comments), then the compiler checks and translates it to assembly code, the assembler converts that into object code (.o/.obj), the linker combines object code with libraries into an executable, and finally the loader loads the executable into memory so the CPU can execute it.

The preprocessor runs before compilation and handles directives like #include, #define, and conditional compilation directives (#ifdef, #ifndef, #if). It also strips out comments before the compiler ever sees the code.

Compilation and assembling only produce object code for your own source file, which may still reference library functions like printf() that live in the Standard C Library. The linker is required to resolve those references and combine everything into one complete executable.

Object code is machine code generated from one source file but is not yet complete — it may have unresolved library references. Executable code is generated after the linker resolves those references and combines everything, producing a complete program the operating system can run.

A compile-time (compilation) error occurs before the program runs, such as a missing semicolon — the executable is never created. A runtime error occurs after successful compilation, while the program is running, such as division by zero, which can cause a crash or undefined behavior depending on the system.
