Pseudocode

Once you've broken a problem into numbered steps, pseudocode is where those steps start looking like code — without yet worrying about C's exact punctuation rules.

1. What Is It?

Pseudocode is an algorithm written in structured, code-like plain English — using keywords such as START, INPUT, IF...THEN...ELSE, WHILE, DISPLAY, and END — without following any real programming language's exact syntax rules.

It's a bridge between the numbered-step algorithms you saw in the previous topic and the real C code you'll start writing in the next chapter.

2. Why Do We Need to Know This?

Pseudocode lets you plan a program's logic without worrying about C's punctuation rules (semicolons, braces, exact keyword names) yet. It separates "getting the logic right" from "getting the syntax right" — a separation that's especially valuable for beginners, who tend to get stuck arguing with syntax errors before their logic is even solid.

3. Real-Life Example

Think of a movie script's stage directions: "ENTER: Character walks to the door, opens it, exits." It's structured and precise, but it isn't written in a formal legal or technical language — anyone can read and follow it. Pseudocode applies that same idea to programming logic: precise enough to act on, plain enough for anyone to read.

5. Syntax

This course uses a small, consistent set of pseudocode keywords and conventions:

  • START / END — mark the beginning and end of the whole pseudocode.
  • INPUT / DISPLAY — read a value in / show a value out.
  • IF...THEN...ELSE...ENDIF — make a decision between two paths.
  • WHILE...ENDWHILE — repeat while a condition stays true.
  • FOR...ENDFOR — repeat a known number of times.
  • Indentation — used to visually show nesting, the same way braces will in C.

6. Step-by-Step Code

Here is the recurring "largest of two numbers" problem, now written as pseudocode:

pseudocode 
START
INPUT A, B
IF A > B THEN
  DISPLAY A
ELSE
  DISPLAY B
ENDIF
END

START marks the beginning of the pseudocode. INPUT A, B reads in the two numbers to compare. IF A > B THEN begins the decision — if it's true, DISPLAY A shows A as the larger value. ELSE handles the case where it's false, so DISPLAY B shows B instead. ENDIF closes the decision block, and END marks the end of the pseudocode.

8. Common Mistakes

  • Forgetting the closing keyword — leaving off ENDIF or ENDWHILE so it's unclear where a block actually ends.
  • Mixing up pseudocode with real syntax, like adding semicolons that don't belong in pseudocode at all.
  • Writing pseudocode so vague ("do the thing with the numbers") that it doesn't actually resolve the logic.

9. Debugging Tips

Read your pseudocode aloud as plain instructions to a friend. If it doesn't make unambiguous sense out loud — if you'd have to stop and explain what you meant — it needs to be more precise before you move on to C.

10. Best Practices

  • Keep indentation consistent so nested logic is easy to see at a glance.
  • Use the same keyword set consistently — don't invent new keywords partway through.
  • Always close what you open: every IF gets an ENDIF, every WHILE gets an ENDWHILE.

15. FAQs

Q: Is pseudocode checked by a compiler?
A: No — it's for humans only, and it is never run by a computer.

Q: Do all courses/companies use the exact same pseudocode keywords?
A: No — conventions vary from place to place. The important part is being unambiguous and consistent within one document.

16. Key Takeaways

  • Pseudocode separates a program's logic from its exact syntax.
  • It uses a small, consistent keyword set — START/END, INPUT/DISPLAY, IF...ENDIF, WHILE...ENDWHILE, FOR...ENDFOR.
  • It's the last planning step before real C code starts in the next chapter.
