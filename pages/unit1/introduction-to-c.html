Introduction to C Program and C Compilers

Almost every interview starts with questions like "What is C?", "Who developed C?", "Why is C still used today?", and "What is a compiler?" Let's understand everything from the beginning.

What is C?

Definition: C is a general-purpose, procedural, high-level programming language used to develop software, operating systems, embedded systems, and many other applications.

In simple words: C is a language used to communicate with a computer to create programs.

Why Was C Developed?

Before C, programmers mainly used languages like Assembly and B (BCPL-based language). These languages had limitations:

  • Difficult to write large programs.
  • Less portable.
  • Hard to maintain.

To solve these problems, Dennis Ritchie developed C.

Who Developed C?

Dennis Ritchie

  • Developed C in 1972.
  • Worked at Bell Laboratories (Bell Labs).
  • Also played a major role in developing the UNIX operating system.

Interview Tip: Father of C = Dennis Ritchie

Why Was C Named "C"?

The language B already existed. Dennis Ritchie improved B and named the new language C, the next letter in the English alphabet.

C

Why is C So Popular?

Because C is:

  • Fast
  • Efficient
  • Portable
  • Powerful
  • Close to hardware
  • Easy to learn compared to Assembly

Even after 50+ years, C is still widely used.

Features of C

Let's understand each feature.

1. Simple

C has a relatively small set of keywords and straightforward syntax.

Example 
int a = 10;
printf("%d", a);

Easy to understand.

2. Fast

C programs compile into efficient machine code. That's why operating systems and games often use C.

3. Portable

A C program written on Windows can often be compiled on Linux or macOS with little or no modification.

Windows
Linux
macOS

This is called portability.

4. Structured Language

Programs are divided into functions.

Example

}

}

This makes programs organized and easier to maintain.

5. Rich Library

C provides many built-in functions.

Examples

scanf()
strlen()
sqrt()
pow()

These come from standard libraries.

6. Middle-Level Language

This is one of the most asked interview questions.

Why is C called a Middle-Level Language?

Because it supports both:

High-Level Features

Example 
printf("Hello");

Easy for humans to understand.

Low-Level Features

  • Pointers
  • Memory Addresses
  • Bitwise Operations

These allow direct interaction with hardware. Therefore, C combines features of both high-level and low-level languages.

7. Case Sensitive

C distinguishes between uppercase and lowercase letters.

Example 
int age;

int Age;

These are two different variables.

C is used in many areas.

Operating Systems

What is a Compiler?

Now let's understand one of the most important concepts.

Definition: A compiler is a software program that converts C source code into machine code so that the computer can execute it.

Simply put: Compiler = Translator

Real-Life Example

Suppose: You know English. Your friend knows only Japanese. You need a translator.

Why Do We Need a Compiler?

The CPU understands only machine language (binary instructions). Humans write programs in C. The compiler converts C code into machine code. Without a compiler, the computer cannot execute a C program.

Intermediate

BCPL was developed by Martin Richards in 1967. Ken Thompson developed B, based on BCPL, in 1970. Dennis Ritchie then improved B and named the result C, the next letter in the alphabet, in 1972.

C is fast, efficient, portable, and close to hardware, which keeps it essential for operating systems, embedded systems, device drivers, and other performance-critical software that newer, higher-level languages aren't as well suited for.

Assembly is machine-dependent and very difficult to write and maintain for large programs. C is portable across machines, easier to read and write thanks to its structured, high-level syntax, while still allowing low-level access via pointers when needed.

Since the CPU only understands machine language, the compiler translates the C source code into machine code first. Only after this translation can the program actually run on the computer.

A C program written on one platform, like Windows, can often be compiled on Linux or macOS with little or no modification, because C compilers exist for nearly every platform and translate the same standard source code accordingly.
