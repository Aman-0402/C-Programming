Execution and Debugging

Writing code is only 50% of programming. The other 50% is running the program correctly, finding mistakes, and fixing mistakes. This process is called Debugging.

What is Program Execution?

Definition: Program Execution is the process of running a compiled program so that the CPU performs the instructions written in the code.

Simply put: Execution = The computer actually performs the tasks written in your program.

Real-Life Example

Imagine you have a recipe for making tea.

Boil Water
Add Tea
Add Milk
Serve

Writing the recipe is like writing a C program. Actually making the tea is like executing the program.

Program Life Cycle

A C program goes through these stages:

Write Code (.c)
Compile
Executable File
Load into Memory
Execution Begins
Program Ends

How Does a Program Execute?

Suppose we have:

hello.c 
#include <stdio.h>

int main()
{
    printf("Hello");

    return 0;
}

Execution order:

Program starts
main() is called
printf() executes
Hello printed
return 0;
Program Ends

Output: Hello

What Happens Inside Memory?

When the program starts, the operating system loads it into RAM. Memory is allocated for variables, functions, and instructions.

Example 
int a = 10;
int b = 20;

Memory (RAM):

a
10
b
20

The CPU reads these values during execution.

What is Debugging?

Definition: Debugging is the process of finding, identifying, and fixing errors (bugs) in a program. A programmer spends a significant amount of time debugging.

Why is it Called a "Bug"?

The term bug became famous in 1947 when a moth was found trapped inside the hardware of the Harvard Mark II computer, causing it to malfunction.

Since then: Error = Bug. Removing the error = Debugging.

Types of Errors

There are three major types of errors every beginner should know: Syntax Errors, Runtime Errors, and Logical Errors. Let's understand each one.

Syntax Error

Definition: A Syntax Error occurs when the rules (syntax) of the C language are violated. The compiler detects these errors.

Example 1: Missing Semicolon

broken.c 
#include <stdio.h>

int main()
{
    printf("Hello")
    return 0;
}

Error: Expected ';'

Reason: Semicolon is missing after printf().

Example 2: Missing Bracket

broken.c 
int main(
{
}

Compiler error because the closing ) is missing.

Characteristics

  • Found during compilation.
  • Program will not run.
  • Easy to fix after reading compiler messages.

Runtime Error

Definition: A Runtime Error occurs after successful compilation, while the program is executing.

Example 
#include <stdio.h>

int main()
{
    int a = 10;
    int b = 0;

    printf("%d", a / b);

    return 0;
}

Problem: 10 ÷ 0. Division by zero is invalid.

Possible result: Runtime Error, or undefined behavior depending on the system.

Other Runtime Errors

  • Division by zero.
  • Accessing invalid memory.
  • File not found.
  • Stack overflow.
  • Null pointer access.

Logical Error

Definition: A Logical Error means the program runs successfully, but the output is incorrect because the logic is wrong. These errors are not detected by the compiler.

Suppose you want to add two numbers.

Correct: sum = a + b;

Wrong: sum = a - b;

Example 
int a = 20;
int b = 10;

sum = a - b;

Output: 10

Expected: 30

The program runs, but the answer is wrong.

Error Comparison

Error TypeDetected ByProgram Runs?
Syntax ErrorCompiler❌ No
Runtime ErrorDuring execution⚠️ Starts but may crash or behave unexpectedly
Logical ErrorProgrammer/Testing✅ Yes, but incorrect output

What is a Debugger?

A Debugger is a software tool that helps programmers:

  • Run programs step by step.
  • Pause execution.
  • Inspect variables.
  • Find the exact line where an error occurs.

Popular IDEs like Code::Blocks, Visual Studio, CLion, and VS Code provide debuggers.

Breakpoint

A Breakpoint tells the debugger: "Pause execution at this line."

Example 
int a = 10;
int b = 20;

int sum = a + b;   ← Breakpoint

printf("%d", sum);

The program pauses before executing the marked line, allowing you to inspect variable values.

Step Into, Step Over, Step Out

These are common debugger controls.

Step Into

Go inside a function.

Example 
sum();

The debugger enters the sum() function.

Step Over

Execute the current line without entering any function calls.

Step Out

Finish the current function and return to the caller.

Watch Window

A Watch Window shows the current values of variables during debugging.

Example 
int a = 10;
int b = 20;

Watch Window: a = 10, b = 20

After:

Example 
a = a + 5;

Watch Window: a = 15, b = 20

This helps you verify that your program behaves as expected.

Tips for Effective Debugging

  1. Read compiler error messages carefully.
  2. Fix the first error first — later errors may disappear automatically.
  3. Test with small inputs.
  4. Use meaningful variable names.
  5. Print variable values using printf() if a debugger is unavailable.
  6. Check loops and conditions carefully.
  7. Verify formulas and calculations.

Common Beginner Mistakes

Example 
printf("Hello")

❌ Missing semicolon.

Example 
int a;

printf("%d", b);

❌ Using an undeclared variable.

Example 
if(a=5)

❌ Assignment instead of comparison. Correct: if(a==5)

Example 
int x = 10/0;

❌ Division by zero.

Interesting Facts 💡

  • Professional developers often spend more time debugging than writing new code.
  • Good debugging skills are often more valuable than typing code quickly.
  • Modern IDEs include advanced debugging tools such as memory inspection, call stacks, and conditional breakpoints.

Quick Revision

  • Execution is the process of running a compiled program.
  • Debugging means finding and fixing bugs.
  • Syntax Errors are detected by the compiler.
  • Runtime Errors occur while the program is running.
  • Logical Errors produce incorrect output even though the program executes.
  • A Debugger helps inspect program execution step by step.

MCQs

Interview Questions

Beginner

Program execution is the process of running a compiled program so that the CPU performs the instructions written in the code.

Debugging is the process of finding, identifying, and fixing errors (bugs) in a program.

Compilation converts source code into machine code and happens first. Execution is running that compiled program so the CPU actually performs its instructions, and happens after compilation and loading.

A syntax error occurs when the rules of the C language are violated, and is detected by the compiler. Example: printf("Hello") missing its semicolon.

A logical error means the program runs successfully but produces incorrect output because the logic is wrong. Example: writing sum = a - b; when you meant sum = a + b;.

Intermediate

Syntax errors (e.g. a missing semicolon) are caught by the compiler and stop the program from running at all. Runtime errors (e.g. division by zero) occur after successful compilation, while the program executes, and may crash it. Logical errors (e.g. subtracting instead of adding) let the program run to completion but produce incorrect output.

A debugger is a software tool that lets programmers run a program step by step, pause execution, inspect variables, and find the exact line where an error occurs — much faster than guessing from output alone.

A breakpoint tells the debugger to pause execution at a specific line, so you can inspect the program's state at that point. A watch window shows the current values of chosen variables during debugging, updating as you step through the code.

Since it's likely a logical error, I'd trace through the code with a debugger or printf statements, checking variable values and formulas at each step against what I expect, to find where the actual behavior diverges from the intended logic.

A single early mistake, like a missing semicolon or bracket, can confuse the compiler about everything that follows, producing a cascade of misleading follow-on errors that often disappear once the real, first error is fixed.
