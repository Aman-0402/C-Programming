Flowcharts

If an algorithm explains the steps in words, a flowchart explains the same steps using symbols and arrows. Algorithm = Text. Flowchart = Picture.

What is a Flowchart?

Definition: A Flowchart is a graphical representation of an algorithm using standard symbols connected by arrows to show the flow of execution.

Simply put: Flowchart = Diagram of a program's logic.

Why Do We Use Flowcharts?

Suppose you explain a route to your friend.

Method 1: You write:

Directions 
Go straight.
Turn left.
Walk 100 meters.
Turn right.

This is like an algorithm.

Method 2: You draw a map. That is like a flowchart.

Pictures are easier to understand than long text.

Why are Flowcharts Important?

They help programmers to:

  • Understand program logic.
  • Find mistakes before coding.
  • Explain programs to others.
  • Plan software development.
  • Convert logic into code easily.

Standard Flowchart Symbols

Every flowchart uses standard symbols.

1. Terminator (Start/Stop)

Represents the beginning or end of a program.

Every flowchart must have one START and one STOP.

START
STOP

2. Process Symbol

Represents calculations or processing.

Sum = A + B

Other examples:

  • Area = L × B
  • Average = Total / 5
  • Salary = Salary + Bonus

3. Input / Output Symbol

Used for reading input or displaying output. In this course's diagrams, input/output steps are shown in the same process box, labeled clearly:

Input A, Input B
Output Sum

4. Decision Symbol

Used for decision making. Usually asks a Yes/No question.

Is Number Even?

Possible outputs: Yes, No.

5. Flow Line

Shows the direction of execution. Without arrows, we don't know what executes next.

6. Connector

Used when a flowchart becomes too large. Instead of drawing long arrows, we connect matching labeled circles (for example, two boxes both marked "A") across the page.

Problem: Find the sum of two numbers.

Algorithm

Advantages of Flowcharts

  • ✅ Easy to understand
  • ✅ Visual representation
  • ✅ Easy debugging
  • ✅ Better communication
  • ✅ Helps before coding

Disadvantages

  • ❌ Large programs produce huge flowcharts.
  • ❌ Updating them takes time.
  • ❌ Complex systems become difficult to draw.

Interesting Facts 💡

  • The first structured flowchart techniques became popular in the 1940s and 1950s for documenting computer programs.
  • Before writing code, many software companies prepare flowcharts during the design phase.
  • Even today, flowcharts are widely used in software engineering, business processes, and manufacturing workflows.

Quick Revision

  • A flowchart is a graphical representation of an algorithm.
  • It uses standard symbols to represent different operations.
  • Arrows indicate the direction of execution.
  • Decision symbols are used for Yes/No conditions.
  • Flowcharts make complex logic easier to understand.

MCQs

Interview Questions

Beginner

A flowchart is a graphical representation of an algorithm using standard symbols connected by arrows to show the flow of execution.

Flowcharts help programmers understand program logic, find mistakes before coding, explain programs to others, plan software development, and convert logic into code easily.

Oval (Terminator), Rectangle (Process), Parallelogram (Input/Output), Diamond (Decision), Arrow (Flow Line), and Circle (Connector).

The diamond symbol is used for decision making, usually asking a Yes/No question.

A flow line (arrow) shows the direction of execution — without it, we wouldn't know what executes next.

Intermediate

An algorithm uses plain English numbered steps and is text-based. Pseudocode uses programming-like keywords and is also text-based, but more detailed and easier to convert into code. A flowchart uses symbols and arrows and is visual, making it easy to explain.

They let programmers visualize the logic, catch missing decision branches or mistakes early, and communicate the design to others before any code is written.

Advantages: easy to understand, visual, easy debugging, better communication, helps before coding. Disadvantages: large programs produce huge flowcharts, updating them takes time, and complex systems become difficult to draw.

Connectors let you link parts of a flowchart using matching labeled symbols instead of drawing long arrows across the page, keeping large flowcharts readable and manageable.
