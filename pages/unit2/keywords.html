Keywords

Words the C language has already claimed for itself — like int, if, and return — that you can never borrow for your own variable or function names.

What Is It?

A keyword is a word reserved by the C language itself for a specific, fixed purpose — words like int, if, return, for, and while. Keywords cannot be used as identifiers (names you choose for your own variables, functions, and so on) because the compiler always interprets them as their reserved meaning, never as something you defined.

Why Do We Need to Know This?

Recognizing keywords by sight — and knowing up front that you can't reuse them as your own names — prevents a whole category of confusing beginner naming errors before they ever happen.

Real-Life Example

Think of reserved words in a legal contract template, like "Party" or "Agreement" — they always mean one specific thing in that context. You can't casually redefine what "Agreement" means partway through the same document; the word is locked to its fixed meaning. C's keywords work the same way.

Visual Explanation

int

A data type keyword — it tells the compiler that a variable will hold a whole number.

return

A keyword that ends a function, optionally sending a value back to whoever called it.

main is NOT a keyword

Surprising but true: main is just a conventionally-named identifier that the operating system/runtime looks for as the program's entry point. It is not reserved by C's grammar at all.

Common Mistakes

  • Trying to name a variable the same as a keyword, e.g. int int = 5; — this is invalid, since int is reserved and the compiler will never treat it as a variable name.
  • Assuming main is a keyword. It isn't — it's just a specially-recognized identifier by convention, not a reserved word in C's grammar.

Debugging Tips

If the compiler reports an error pointing at what looks like a perfectly normal word you tried to use as a variable name, check whether that word happens to be a reserved keyword — that mismatch is a very common source of confusing beginner errors.

Best Practices

  • Avoid keyword-adjacent names that could confuse readers. Naming a variable printf_result is perfectly fine, but don't get creative trying to almost-reuse a keyword itself.

FAQs

Q: How many keywords does C have?
A: A relatively small, fixed set — roughly 32 in standard C, though this course doesn't require memorizing the full list up front.

Q: Are keywords case-sensitive?
A: Yes — int is a keyword, but Int or INT are not, since C is case-sensitive throughout.

Key Takeaways

  • A keyword is a word reserved by the C language itself for a specific, fixed purpose, like int or return.
  • Keywords can never be reused as identifiers — the compiler always interprets them with their fixed, built-in meaning, so reusing one causes a compiler error.
  • main is not a keyword; it's just a conventionally-named identifier the OS looks for as the program's entry point, not a reserved word in C's grammar.
