Keywords in C

A keyword is a reserved word in the C language that has a predefined meaning to the compiler. You cannot use a keyword as the name of a variable, function, or any identifier.

What are Keywords?

Definition: Keywords are reserved words predefined by the C language that have special meanings and cannot be used as identifiers.

Real-Life Analogy

Imagine a classroom. The teacher says: Present, Absent, Leave. These words have fixed meanings. You cannot suddenly decide that "Present" means "Go Home."

Similarly, in C, int always means declare an integer variable. You cannot change its meaning.

Why Do We Need Keywords?

Without keywords, the compiler wouldn't know what you want to do.

Example 
int age = 20;

Compiler understands: int → Create an integer variable, age → Variable name, 20 → Value.

Without int, the compiler wouldn't know the data type.

Number of Keywords in C

The original ANSI C standard defines 32 keywords. Modern C standards (C99, C11, C18, C23) add more keywords, but the classic 32 keywords are the most important and are commonly taught in introductory courses.

Rules for Keywords

Rule 1: Keywords are reserved.

❌ Wrong: int int = 10; — Compiler Error

Rule 2: Cannot use keywords as variable names.

❌ Wrong: float return = 5.5;

Rule 3: Cannot use keywords as function names.

Wrong 
void while()
{
}

Rule 4: Keywords are lowercase.

Correct: int — Wrong: Int

Int is treated as an identifier, not the int keyword.

Common Beginner Mistakes

Mistake 1: int if = 5; — Wrong because if is a keyword.

Mistake 2: float while = 2.5; — Wrong.

Mistake 3: Thinking keywords are case-insensitive.

Wrong: INT age; — Correct: int age;

C is case-sensitive.

Keywords vs Identifiers

KeywordIdentifier
Reserved by CCreated by programmer
Fixed meaningUser-defined meaning
Cannot be used as variable namesCan be variable/function names
Examples: int, if, whileExamples: age, marks, salary

Interesting Facts

Fact 1: Every C compiler has a predefined list of keywords.

Fact 2: Keywords are recognized during the lexical analysis (tokenization) phase of compilation.

Fact 3: The number of keywords has increased in newer C standards, but the original 32 remain the foundation.

Fact 4: You cannot redefine or overload a keyword in C.

Complete Example

complete.c 
#include <stdio.h>

int main()
{
    const int PASS = 40;

    int marks = 70;

    if(marks >= PASS)
    {
        printf("Pass");
    }
    else
    {
        printf("Fail");
    }

    return 0;
}

Keywords used: const, int, if, else, return

Output: Pass

Summary

  • Keywords are reserved words with predefined meanings.
  • You cannot use keywords as identifiers.
  • ANSI C has 32 standard keywords.
  • Keywords are case-sensitive.
  • Grouping keywords by purpose makes them easier to learn.

Because the compiler always interprets a keyword using its fixed, reserved meaning, so reusing it as a variable name would create an unresolvable conflict.

32 keywords, as defined by the original ANSI C standard. Later standards (C99, C11, C18, C23) add more.

A keyword is reserved by C with a fixed meaning and cannot be used as a variable name (e.g. int, if, while). An identifier is a name created by the programmer with user-defined meaning (e.g. age, marks, salary).

Yes. int is a keyword, but INT or Int are not — C is case-sensitive throughout, so Int age; treats Int as an ordinary identifier, not the int keyword.
