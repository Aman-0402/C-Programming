Operators

A symbol that tells the compiler to perform a specific operation — this page maps out the categories of operators you're about to meet, before the next three pages go deep on each one.

What Is It?

An operator is a symbol that tells the compiler to perform a specific operation on one or more values, called operands — like + for addition or > for "greater than." C groups its operators into several categories: arithmetic operators (for math), relational operators (for comparison), logical operators (for combining true/false conditions), and others — such as assignment, bitwise, and conditional operators — covered in a later chapter of this course.

Why Do We Need to Know This?

Seeing the operator categories laid out together, before diving into each one individually, gives you a clear map of what's coming and how the categories relate to each other. Instead of meeting dozens of symbols with no context, you'll know upfront that arithmetic operators calculate, relational operators compare, and logical operators combine conditions.

Real-Life Example

Think of different tool categories in a toolbox: measuring tools (arithmetic, for calculating totals and quantities), comparison tools (relational, for checking which of two things is bigger, smaller, or equal), and decision tools (logical, for combining multiple yes/no conditions into a single answer). Each category solves a fundamentally different kind of problem, just like each toolbox category serves a different job.

Logical Operators

&& || ! — combine multiple true/false conditions together. Covered three pages ahead.

Beyond these three, C also has assignment, bitwise, and conditional operators (among others) — these exist and are useful, but they're covered in a later chapter of this course rather than here.

Common Mistakes

  • Assuming all operators work the same way regardless of category — arithmetic operators return calculated numeric values, while relational and logical operators are fundamentally about true/false-style results, a different kind of outcome entirely.
  • Trying to memorize every operator's symbol without first understanding its category's underlying purpose — this makes the symbols harder to recall and easy to mix up.

Debugging Tips

When an expression's result doesn't make sense, first identify which operator category is involved. Arithmetic bugs are usually about wrong calculations (a miscalculated total, an unexpected result from division). Relational and logical bugs are usually about faulty true/false logic (a condition that's true when it shouldn't be, or vice versa). Narrowing down the category first makes the actual bug much faster to spot.

Best Practices

  • Learn operators by category, understanding what kind of question or task each category answers — calculating, comparing, or combining conditions — rather than memorizing symbols in isolation.

FAQs

Q: Are there more operator categories than just these three?
A: Yes — assignment, bitwise, conditional, and others exist, covered in a later chapter of this course.

Q: Do all operators in C use symbols, or are some written as words?
A: The operators covered in this course's next few pages are symbol-based. C does have some keyword-based constructs, but they aren't classified as operators in the same sense.
