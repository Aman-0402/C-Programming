Floating Point Types

Not everything is a whole number — this page adds a decimal-valued height to the growing student profile example, and introduces float (and its more precise cousin, double).

What Is It?

A floating-point type (float or the more precise double) stores numbers that can have a decimal point, like 5.9 or 3.14. Unlike int, floating-point types can represent fractional values, not just whole numbers.

Why Do We Need to Know This?

Many real-world quantities — height, price, temperature — naturally have decimal precision. Using the right type for these values avoids the truncation problem introduced on the previous page, where forcing a decimal value into an int silently drops everything after the decimal point.

Real-Life Example

Think of the difference between a container that only counts whole items and a measuring tape marked in fine increments (fractions of an inch). A floating-point type is the fine-grained measuring tool — built for values that aren't naturally whole numbers.

Visual Explanation

Extending the previous page's memory diagram, here's the student profile with its new height variable sitting alongside age:

variable name: age (int)
value: 20 — whole number
variable name: height (float)
value: 5.9 — decimal value

age holds a whole number and is declared as int. height holds a decimal value and needs a floating-point type — that's exactly what float is for.

Syntax

syntax 
float variableName = decimalValue;

// example:
float height = 5.9;

double also exists in C, for higher precision — it stores more decimal digits accurately than float, and is common when precision matters more. For many everyday cases, though, float is perfectly sufficient.

Step-by-Step Code

Extending this unit's profile.c example with the new height variable:

profile.c 
#include <stdio.h>

int main() {
    int age = 20;
    float height = 5.9;

    printf("Age: %d\n", age);
    printf("Height: %.1f\n", height);

    return 0;
}

height is declared as float since it needs to hold a decimal value, unlike age. The printf format specifier for a float is %f — here, %.1f is used to show exactly 1 decimal place. Compare this with %d, used for the whole-number age: the format specifier always has to match the variable's actual type.

Common Mistakes

  • Using %d to print a float value. This produces garbage or incorrect output — the format specifier must match the variable's actual type.
  • Assuming float is perfectly precise for all decimal values. Floating-point representation actually has subtle precision limitations — a deeper topic beyond this course's current scope, but worth knowing exists.

Debugging Tips

If your printf output for a decimal variable looks like garbage or completely wrong, double-check that you used %f (or %.Nf) rather than %d — mismatched format specifiers are a very common source of confusing output.

Best Practices

  • Always match your printf format specifier to the variable's actual type: %d for int, %f/%.Nf for float/double.
  • Use %.Nf to control exactly how many decimal places are displayed, for cleaner, more predictable output.

FAQs

Q: Should I always use double instead of float?
A: Not necessarily — float suffices for many everyday needs; double is preferred when higher precision genuinely matters, a nuance covered more in later material.

Q: Can I print a float without any decimal places?
A: Yes, technically, but it's more common and useful to control the exact number of decimal places shown with %.Nf.

Key Takeaways
