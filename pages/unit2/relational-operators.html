Relational Operators

Comparing two values to get a true/false-style answer — the foundation of every decision a program will ever make.

What Is It?

Relational operators compare two values and produce a result representing true or false. C provides six of them: > (greater than), < (less than), >= (greater than or equal to), <= (less than or equal to), == (equal to), and != (not equal to). In C, a true result is represented as the integer 1, and a false result as 0 — there's no separate dedicated boolean type used here, which is a nuance worth keeping in mind.

Why Do We Need to Know This?

Relational operators are the foundation of decision-making in programs — comparing values is how a program can behave differently depending on its data. This idea will be explored fully once control statements are introduced in a later unit, but the comparisons themselves start here.

Real-Life Example

Think of a bouncer at a door checking "is this person old enough?" The check itself produces a simple yes/no (true/false) answer, and that answer is what determines what happens next — let them in, or turn them away.

Syntax

syntax 
result = operand1 relationalOperator operand2;

// example:
int isGreater = a > b;

The result is stored as an int: 1 for true, 0 for false.

Step-by-Step Code

Extending add.c with a relational comparison:

add.c 
#include <stdio.h>

int main() {
    int a = 5;
    int b = 3;
    int isGreater = (a > b);

    printf("Is a greater than b? %d\n", isGreater);

    return 0;
}
  • int a = 5; and int b = 3; declare the two integers being compared, just as in the original add.c.
  • int isGreater = (a > b); evaluates to true since 5 is greater than 3, so isGreater is set to 1.
  • printf("Is a greater than b? %d\n", isGreater); prints isGreater using %d, since the comparison result is stored as an int under the hood, not a special boolean type.

Dry Run

Tracing through execution with a = 5, b = 3:

  • a > b evaluates to true, since 5 is indeed greater than 3.
  • isGreater becomes 1.

The exact final output is:

output 
Is a greater than b? 1

If a and b were swapped instead (a = 3, b = 5), the comparison a > b would be false, so isGreater would become 0, and the program would print Is a greater than b? 0.

Common Mistakes

  • Confusing = (assignment) with == (equality comparison) — a very common and easy-to-make beginner mistake, since a = b assigns b's value to a, while a == b only checks whether they're equal, without changing anything.
  • Expecting a special "true"/"false" word to print instead of the numeric 1/0 that C actually produces.

Debugging Tips

If a comparison seems to be silently changing a variable's value instead of just checking it, look closely for an accidental single = where you meant the double ==.

Best Practices

  • Always double-check == versus = when writing a comparison.
  • Some programmers deliberately write comparisons with the literal/constant first (e.g. 5 == a instead of a == 5), specifically so that an accidental single = typo causes an obvious compiler error rather than a silent bug — a well-known defensive habit worth knowing about.

FAQs

Q: Why does C print 1 instead of the word "true"?
A: Because relational operator results are plain ints in this context, not a distinct printable boolean type — 1 and 0 are the actual stored values.

Q: Is == always safe to use for comparing any two values in C?
A: For the numeric types covered so far, yes. Comparing entire strings requires a different approach, a topic covered later in this course.

Key Takeaways

  • C provides six relational operators: >, <, >=, <=, ==, and !=.
  • Relational comparisons in C produce a plain int result: 1 for true, 0 for false — there's no separate boolean type in play here.
  • Confusing = (assignment) with == (comparison) is a classic and dangerous C mistake — always double-check which one you meant.
