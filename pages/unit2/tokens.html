Tokens

Welcome to Unit 2 — before you can talk about variables, data types, or operators, you need to understand the smallest meaningful pieces the compiler actually sees: tokens.

What Is It?

A token is the smallest individual meaningful unit in a C program. Before the compiler does anything else with your source code, it breaks it down into a stream of tokens — this is the very first step of compilation.

C has several categories of tokens:

  • Keywords — reserved words like int, return, if.
  • Identifiers — names given to variables, functions, etc., like sum or main.
  • Constants — fixed values like 5 or 3.14.
  • Strings — text enclosed in double quotes, like "Hello".
  • Operators — symbols that perform operations, like + or =.
  • Punctuation / special symbols — things like ;, {, }, and ( ).

Why Do We Need to Know This?

Tokens are the foundation for everything else in this unit. Keywords, identifiers, variables, and operators are all specific kinds of tokens — so understanding this umbrella concept first makes each individual topic click faster as you meet it.

Real-Life Example

Think of breaking a sentence down into its individual words and punctuation marks before analyzing its grammar. You can't understand a sentence's structure without first recognizing its smallest meaningful pieces — a C compiler works the same way with your code.

Visual Explanation

example.c 
int sum = a + b;

The compiler sees this single line as seven separate tokens:

intkeyword
sumidentifier
=operator
aidentifier
+operator
bidentifier
;punctuation

Common Mistakes

  • Assuming a token must be a single character — many tokens, like keywords (int) and identifiers (sum), are multi-character.
  • Confusing "token" with "line of code" — a single line typically contains many tokens, not just one.

Debugging Tips

The compiler's very first job is breaking your source code into tokens. So when you see a "tokenizing" or "unexpected token" error, it usually points to something like a stray symbol or an unclosed string — the compiler couldn't even recognize a valid token at that spot, let alone check its meaning.

Best Practices

  • Keep code readable by spacing tokens clearly, even though C itself is flexible about whitespace between tokens.
  • Remember that clarity for human readers still matters, even when the compiler doesn't strictly require it.

MCQs

Programming Exercises

Easy: Define "token" in your own words.

Medium: Count the tokens in int a = 5; (hint: there are 5 — int, a, =, 5, ;).

Hard: Break down printf("Hello\n"); into its individual tokens.

Assignments

Pick any line from add.c, such as int sum = a + b;, and list every token in it along with its category (keyword, identifier, operator, or punctuation).

FAQs

Q: Does whitespace count as a token?
A: No — whitespace generally separates tokens, but it isn't a token itself.

Q: Are comments tokens?
A: No — comments are stripped out before tokenizing; they're there for human readers only, not for the compiler.

Key Takeaways

  • A token is the smallest individual meaningful unit the compiler recognizes in your source code.
  • C's main token categories are keywords, identifiers, constants, strings, operators, and punctuation/special symbols.
  • Tokens are the foundation this whole unit builds on — every topic ahead, from keywords to variables to operators, is really just a deeper look at a specific kind of token.
