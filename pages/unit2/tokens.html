C Tokens

A Token is the smallest individual unit (building block) of a C program that has meaning to the compiler.

What are C Tokens?

Think of English. Sentence: "I love programming." Smallest meaningful parts are: I, love, programming, .

Similarly, in C:

Example 
int age = 20;

Compiler divides it into small meaningful pieces:

int
age
=
20
;

Each of these is called a Token.

Definition

A Token is the smallest meaningful element recognized by the C compiler. Everything you write in C is made of tokens.

Why are Tokens Important?

Imagine building a house. House → Bricks. Similarly, Program → Tokens. Without tokens, no program exists.

Example

#include<stdio.h>

Common Beginner Mistakes

Mistake 1: Thinking spaces are tokens.

Example 
int     a     =     10;

Spaces are ignored. Tokens remain: int, a, =, 10, ;

Mistake 2: Thinking 10; is one token.

Wrong: 10; — Correct: 10 then ; (two separate tokens).

Mistake 3: Thinking printf("Hello"); is one token.

Actually: printf, (, "Hello", ), ; (five separate tokens).

Interesting Facts

Fact 1: Every C compiler first converts your code into tokens before checking syntax.

Fact 2: Even a million-line C program is broken into tokens internally.

Fact 3: The compiler cannot understand your program until it identifies all tokens correctly.

Fact 4: Removing spaces usually doesn't affect tokenization.

Example 
int a=5;

and

Example 
int        a      =       5;

Both produce the same tokens.

Summary

  • A token is the smallest meaningful unit of a C program.
  • Every C program is built from tokens.
  • There are 6 major types of tokens: Keywords, Identifiers, Constants, String Literals, Operators, Special Symbols.
  • The compiler first breaks the source code into tokens before parsing it.
