Strings

The last data type in this chapter — a sequence of characters ending with an invisible marker — and the piece that finally completes the profile.c example.

What Is It?

A string in C is a sequence of characters, stored as a character array (char[]) that ends with a special null terminator character, written \0, marking exactly where the string ends. Unlike int, float, or char, C doesn't have a dedicated built-in "string" type — strings are built directly from the char type you just learned, arranged one after another in sequence.

Why Do We Need to Know This?

Strings tie together several concepts already covered in this chapter — the char type, and the idea of a sequence of values in memory — into the single most common way programs handle text. Names, messages, and labels are almost always represented as strings, making this one of the most frequently used pieces of C you'll write.

Real-Life Example

Think of a row of individual letter tiles placed side by side, spelling out a word — like Scrabble tiles lined up to spell "Alex." Right after the last letter tile, there's an invisible "end" marker placed down, so that anyone reading the tiles knows exactly where the word stops and nothing that comes after it is part of the word.

Visual Explanation

Here is the string "Alex" laid out in memory as 5 connected character cells, including its automatic null terminator as the final, invisible-but-essential character:

name[0]
'A'
name[1]
'l'
name[2]
'e'
name[3]
'x'
name[4]
'\0'

The first four cells hold the visible letters. The fifth cell holds the null terminator, '\0' — it isn't printed or displayed, but it's essential: it's how C knows the string's actual content has ended.

Syntax

syntax 
char variableName[] = "text";

// example:
char name[] = "Alex";

The double quotes signal a string literal; C automatically adds the \0 null terminator at the end for you — you never have to type it yourself. The printf format specifier for a string is %s.

Step-by-Step Code

This is the completed profile.c example, now with all 4 data types together:

profile.c 
#include <stdio.h>

int main() {
    int age = 20;
    float height = 5.9;
    char grade = 'A';
    char name[] = "Alex";

    printf("Name: %s\n", name);
    printf("Age: %d\n", age);
    printf("Height: %.1f\n", height);
    printf("Grade: %c\n", grade);

    return 0;
}
  • int age = 20; declares an integer variable holding the whole number 20.
  • float height = 5.9; declares a floating-point variable holding a decimal value.
  • char grade = 'A'; declares a single character, using single quotes.
  • char name[] = "Alex"; declares name with [], making it an array of characters, initialized with a double-quoted string literal — this is what makes it a string rather than a single char.
  • printf("Name: %s\n", name); uses %s to print the whole string, from its first character up to (but not including) the null terminator.
  • The remaining printf lines print age, height, and grade using their matching format specifiers: %d, %.1f, and %c.

Dry Run

Tracing through execution line by line:

  • age is set to 20.
  • height is set to 5.9.
  • grade is set to 'A'.
  • name is set to the character sequence "Alex", with C automatically placing a \0 right after the 'x'.
  • Each printf line then runs in order, substituting the correct format specifier for each variable's type: %s for name, %d for age, %.1f for height, and %c for grade.

The exact final output is:

output 
Name: Alex
Age: 20
Height: 5.9
Grade: A

Common Mistakes

  • Using single quotes instead of double quotes for a string — 'Alex' is invalid, since single quotes are reserved for single char literals, not multi-character strings.
  • Using the wrong format specifier — printing a string with %d or %c instead of %s produces incorrect (often garbled) output.
  • Forgetting that C doesn't have a dedicated built-in string type — a string is really a char array under the hood, which matters more once string manipulation functions are covered later in this course.

Debugging Tips

If printf output for a name or text variable looks garbled or wrong, double-check two things: that you used %s (not %d or %c), and that the variable was actually declared as a char array initialized with double-quoted text.

Best Practices

  • Always use double quotes for string literals, and single quotes for individual char literals — mixing these up is a common source of compiler errors.
  • Match printf format specifiers carefully to each variable's actual type — a theme that has recurred across this entire chapter: %d for int, %f for float, %c for char, and %s for strings.

FAQs

Q: Is a string the same as a char array?
A: Essentially yes, in C — a string literal is stored as a char array with an automatic null terminator appended at the end.

Q: Can strings be compared or combined directly with operators like + in C?
A: No, not directly with +, unlike some other languages — string manipulation in C requires specific functions, a topic covered later in this course.
