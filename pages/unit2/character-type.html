Character Type

C's type for storing a single character — and the next piece of profile.c, right before strings tie it all together.

What Is It?

char is C's type for storing a single character — a letter, digit, symbol, or punctuation mark. Character literals are written in source code with single quotes, for example 'A'. Internally, a char is actually stored as a small integer (its character code), but it's used and displayed as a character.

Why Do We Need to Know This?

Understanding char sets up Strings, the next page in this chapter — a string in C is fundamentally built from a sequence of individual characters, so getting comfortable with a single char first makes strings much easier to understand.

Real-Life Example

Think of a single printed letter tile in a board game — one letter per tile. A char holds exactly one character, no more, just like one tile holds exactly one letter.

Visual Explanation

char grade
'A'

The variable grade is declared as a char, holding a single character: 'A'.

Syntax

syntax 
char variableName = 'singleCharacter';

A char literal uses single quotes, with exactly one character inside them. This is worth calling out explicitly, because it's a common early point of confusion: single quotes ('A') are for a single char, while double quotes ("A") are for a string, covered on the next page. They look similar but are entirely different types.

Step-by-Step Code

Continuing this chapter's profile.c example, we now add a grade variable:

profile.c 
#include <stdio.h>

int main() {
    int age = 20;
    float height = 5.9;
    char grade = 'A';

    printf("Age: %d\n", age);
    printf("Height: %.1f\n", height);
    printf("Grade: %c\n", grade);

    return 0;
}
  • grade is declared as char, holding the single character 'A' — written in single quotes.
  • The printf format specifier for a char is %c, which is why "Grade: %c\n" is used to print grade.
  • age and height from the previous two pages are unchanged, printed with %d and %.1f as before.

Common Mistakes

  • Using double quotes instead of single quotes for a char literal — "A" is actually a string, not a char, causing a type mismatch.
  • Trying to put more than one character in a char literal — 'AB' is invalid, since a char holds exactly one character.
  • Using the wrong printf format specifier — %d would print the character's underlying numeric code instead of the character itself; %c is correct.

Debugging Tips

If a char variable prints as a number instead of the expected letter, check whether you accidentally used %d instead of %c in the printf call — that's the most common cause of this exact symptom.

Best Practices

  • Always use single quotes for char literals and double quotes for strings — keeping this distinction clear from the start avoids a very common beginner mixup.

FAQs

Q: Can a char hold something other than letters, like a digit or symbol?
A: Yes — any single character, including digits like '5' or symbols like '!', can be stored in a char.

Q: Is char the same as a 1-character string?
A: No — they're different types in C; a string (covered next) is a different construct entirely, even a 1-character one.

Key Takeaways

  • A char stores exactly one character, written in single quotes, like 'A'.
  • Single quotes are for a char, double quotes are for a string — a distinction worth keeping straight from the start.
