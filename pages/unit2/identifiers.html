Identifiers

Now that you can tell C's reserved keywords apart from everything else, meet the "everything else": the names you invent yourself for variables, functions, and more.

1. What Is It?

An identifier is the name a programmer chooses for something in a program — a variable, a function, or any other user-defined element. In int sum = a + b;, the words sum, a, and b are all identifiers: names you picked, not words C already reserved.

2. Why Do We Need to Know This?

Being able to tell identifiers apart from keywords clarifies which words in your code are C's fixed vocabulary and which words you invented yourself. That distinction matters both when reading someone else's code — so you know what's built-in versus custom — and when writing your own, so you don't accidentally try to reuse a reserved word as your own name.

3. Real-Life Example

Think of the difference between a person's chosen nickname and a reserved title like "President." A nickname is something an individual picks for themselves — it can be anything they like. A reserved title, on the other hand, always means the same official thing to everyone, and no individual gets to redefine it. Identifiers are like nicknames; keywords are like reserved titles.

4. Visual Explanation

add.c (identifiers highlighted) 
int a = 5;        // "a"   → identifier (a name YOU chose)
int b = 3;        // "b"   → identifier (a name YOU chose)
int sum = a + b;  // "sum" → identifier (a name YOU chose)

In each line above, int is a keyword — it's not something you get to rename or redefine, it always means the same thing to the compiler. But a, b, and sum are identifiers — names that exist only because you chose them. You could have called sum anything else, like total or result, and the program would work exactly the same way.

5. Common Mistakes

  • Confusing identifiers with keywords. They're actually opposites: a keyword is reserved by C and means the same thing everywhere, while an identifier is a name you chose yourself for your own variable, function, or other element.
  • Choosing unclear, non-descriptive identifier names — for example, naming something x when totalScore would communicate its purpose far more clearly. (The full rules and conventions for naming identifiers well are covered on the next page.)

6. Debugging Tips

If you ever see an "undeclared identifier" compiler error, it means you used a name the compiler has never seen defined anywhere. This is almost always caused by a typo in a variable or function name, or by forgetting to declare something before using it.

7. Best Practices

  • Choose identifiers that describe what they represent — even at this early stage, before you've learned the formal naming rules. It's a habit worth building from your very first programs.

13. Key Takeaways

  • An identifier is a name a programmer chooses for something in a program, such as a variable or function.
  • Identifiers and keywords are opposites: keywords are reserved by C and mean the same thing everywhere, while identifiers are names you invent yourself.
  • Choosing descriptive identifier names is a good habit worth building early, even before you've learned the formal naming rules on the next page.
