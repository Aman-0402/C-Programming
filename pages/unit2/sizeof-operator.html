sizeof Operator

How sizeof reports the exact number of bytes a data type or variable occupies in memory — turning the abstract idea of a "memory diagram" into a concrete, measurable number.

What Is It?

sizeof is an operator — not a function, despite the parentheses often used with it — that returns the number of bytes a data type or variable occupies in memory. For example, sizeof(int) might return 4, meaning an int typically takes up 4 bytes on many common systems.

Why Do We Need to Know This?

sizeof connects directly back to earlier chapter content about data types actually occupying real memory. It makes the abstract idea of "memory diagrams" — boxes representing bytes — concrete and measurable, by letting you ask exactly how many bytes any type or variable takes up.

Real-Life Example

Think of asking "how much shelf space does this box take up?" sizeof answers exactly that question for a data type or variable, in bytes — it doesn't tell you what's inside the box, just how much room it occupies.

Syntax

syntax 
sizeof(type)       // e.g. sizeof(int)
sizeof(variable)    // e.g. sizeof(a)

Note: sizeof's exact byte-count results can vary slightly across different systems and compilers, though common modern defaults — like 4 bytes for int — are typical.

Step-by-Step Code

Using sizeof with add.c's variables:

add.c 
#include <stdio.h>

int main() {
    int a = 5;

    printf("Size of int: %lu bytes\n", sizeof(int));
    printf("Size of a: %lu bytes\n", sizeof(a));

    return 0;
}
  • sizeof(int) returns the size of the int type itself.
  • sizeof(a) returns the size of the specific variable a — which is also declared as an int, so it returns the same value.
  • The %lu format specifier is used here since sizeof returns a special unsigned type. This detail is only previewed at this stage, not deeply covered.

Dry Run

Tracing through execution: both sizeof(int) and sizeof(a) typically return 4 on many common systems, since a is declared as int.

The illustrative output (this can vary by system):

output 
Size of int: 4 bytes
Size of a: 4 bytes

Common Mistakes

  • Assuming sizeof's result is always identical across every system or compiler — it can vary, though common defaults are typical on most modern systems.
  • Treating sizeof like a regular function call in all respects — it's actually a compile-time operator, evaluated before the program even runs, not a runtime function call.

Debugging Tips

If you're unsure how much memory a variable or type occupies, sizeof gives you a direct, reliable answer rather than guessing based on assumptions.

Best Practices

  • Use sizeof rather than hardcoding assumed byte sizes when memory size actually matters in your code — a technique that becomes more relevant with dynamic memory allocation, covered much later in this course.

FAQs

Q: Does sizeof work on any data type?
A: Yes — any built-in or user-defined type/variable.

Q: Will sizeof's result always be exactly the same on every computer?
A: Not guaranteed by the language itself, though common values are typical on most modern systems.

Key Takeaways

  • sizeof returns the number of bytes a data type or variable occupies in memory.
  • It's an operator, not a function — despite the parentheses often used with it, it's evaluated at compile time, before the program runs.
  • sizeof connects directly back to earlier data-type and memory-diagram content, making abstract byte counts concrete and measurable.
