Naming Rules

Not every word you might want to use as a name is actually allowed — C imposes specific, exact constraints on what a valid identifier can look like.

1. What Is It?

Naming rules are the specific constraints C imposes on what a valid identifier can look like. They define exactly which characters an identifier can start with, which characters can follow, and which names are off-limits entirely — regardless of how sensible the name might seem to you.

2. Why Do We Need to Know This?

Knowing the exact rules up front prevents confusing "invalid identifier" errors before they happen, rather than debugging them by trial and error after the compiler rejects a name you assumed was fine.

3. Real-Life Example

Like the specific rules for choosing a valid username on a website — it must start with a letter, it can't contain spaces, and certain symbols aren't allowed. Some choices are simply rejected by the system, no matter how sensible they seem to you.

4. Visual Explanation

Must Start With a Letter or Underscore

The first character of an identifier must be a letter (a-z, A-Z) or an underscore (_) — never a digit.

Letters, Digits, Underscores After That

After the first character, an identifier can contain letters, digits (0-9), and underscores in any combination.

Cannot Be a Keyword

An identifier can never be one of C's reserved keywords, like int, return, or for — those are already spoken for.

Case-Sensitive

total and Total are treated as two completely different identifiers in C.

No Spaces or Special Symbols

Symbols like -, @, and plain spaces are never allowed anywhere in an identifier.

5. Syntax

  • The first character must be a letter (a-z, A-Z) or an underscore (_).
  • Subsequent characters can be letters, digits (0-9), or underscores, in any combination.
  • There is no practical length limit for a beginner — real compilers do have internal limits, but they're not a concern at this level.
  • An identifier must not match any C keyword (see the Keywords page).

6. Step-by-Step Code

Here are some valid and invalid identifier examples, side by side, with the reason for each:

naming-rules.c 
int total;        // valid: starts with a letter
int _count;       // valid: starts with underscore
int score2;       // valid: digit allowed after first character
int 2ndPlace;     // INVALID: cannot start with a digit
int my-score;     // INVALID: hyphen is not allowed
int int;          // INVALID: "int" is a reserved keyword

total is valid because it starts with a letter. _count is valid because an underscore is allowed as a first character. score2 is valid because digits are fine after the first character — just not as the first character itself. 2ndPlace is invalid because it starts with a digit, which the syntax rules forbid outright. my-score is invalid because a hyphen isn't a letter, digit, or underscore, so it isn't allowed anywhere in an identifier. And int is invalid as an identifier because it's already a reserved C keyword — the compiler needs that word to mean something else.

7. Common Mistakes

  • Starting an identifier with a digit — 2ndPlace is invalid no matter how natural it reads.
  • Using hyphens or spaces instead of underscores — my-score or my score are both invalid; use my_score or myScore instead.
  • Forgetting that C is case-sensitive — assuming Total and total are the same identifier, when they're actually two distinct ones.

8. Debugging Tips

If the compiler reports an unexpected syntax error right where you declared a variable, double-check your identifier against these rules first — an invalid name is a very common, easy-to-miss cause.

9. Best Practices

  • Stick to letters, digits, and underscores — nothing else.
  • Start with a lowercase letter for variables — a common convention, not a strict rule.
  • Be consistent about case style across your own code — always camelCase, or always snake_case, but not a mix of both.

14. FAQs

Q: Can an identifier contain uppercase and lowercase letters together?
A: Yes — for example, totalScore is a perfectly valid identifier that mixes both cases.

Q: Is there a maximum identifier length?
A: Practically speaking, no concern for this course — real compilers have generous internal limits that are far beyond anything a beginner would run into.

15. Key Takeaways

  • A valid identifier must start with a letter or underscore, and every character after that must be a letter, digit, or underscore — no digits first, and no other symbols or spaces anywhere.
  • C is case-sensitive, so total and Total are treated as two entirely different identifiers.
  • An identifier can never be one of C's reserved keywords — the same keywords covered on the Keywords page are always off-limits as names.
