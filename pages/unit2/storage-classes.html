Storage Classes

Four keywords — auto, static, register, and extern — that control a variable's lifetime, default value, and visibility beyond the basics already covered.

What Is It?

A storage class in C specifies a variable's lifetime (how long it exists in memory), its default initial value, and additional scope/visibility characteristics beyond what's already been covered. C has exactly 4 storage class keywords: auto, register, static, and extern.

Why Do We Need to Know This?

Storage classes control details like whether a variable's value persists between function calls or resets each time. Understanding this prevents confusion about why some variables seem to "remember" their previous value across calls while others don't.

Real-Life Example

Think of the difference between a whiteboard that gets erased after every meeting (a normal/auto local variable, reset each time it's used) versus a sticky note left on a desk that stays put between meetings (a static variable, which persists between calls).

Visual Explanation

The 4 C storage classes, at a glance:

auto

The default for local variables. This course's examples like add.c's a, b, and sum are technically auto even without writing the keyword, since it's implicit.

static

A local variable that keeps its value between function calls, instead of resetting each time the function runs.

register

A hint to the compiler to keep a variable readily accessible for speed — largely a historical/advisory keyword in modern C.

extern

Declares a variable defined elsewhere, often in another file. A preview concept, fully relevant once multi-file programs are covered later in this course.

Syntax

A storage class keyword goes before the type in a declaration:

syntax 
static int count = 0;
register int i;
extern int sharedValue;

auto is the default for local variables and is rarely written explicitly — you could write auto int a = 5;, but virtually no one does.

Common Mistakes

  • Assuming every local variable needs an explicit storage class keyword — auto is implicit and almost never written.
  • Confusing static's effect (persisting a value between calls) with const's effect (making a value permanently unchangeable) — they solve different problems and can even be combined.

Debugging Tips

If a variable's value unexpectedly "remembers" its previous state across separate calls to the same function, check whether it was declared static.

Best Practices

  • Use static deliberately and sparingly, only when you specifically need a value to persist across calls.
  • Overusing static can make code harder to reason about, since the variable's behavior is no longer purely local to one call.

FAQs

Q: Do I need to memorize all 4 storage classes right now?
A: Recognizing auto and static is most important for now; register and extern become more relevant later in this course.

Q: Is auto in C the same as "auto" in some other languages (like C++'s type inference)?
A: No — C's auto is purely about storage class/lifetime, unrelated to type inference in other languages.

Key Takeaways

  • C has 4 storage class keywords: auto, register, static, and extern.
  • auto is the implicit default for local variables and is almost never written explicitly.
  • static makes a local variable's value persist between separate calls to the function, instead of resetting each time.
