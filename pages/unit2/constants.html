Constants

Not every value in a program should be free to change — the const keyword lets you declare a value that, once set, is sealed for the rest of the program.

What Is It?

A constant is a value that cannot be changed after it's set. In C, you declare a variable as constant using the const keyword, which tells the compiler to reject any later attempt to modify it.

Why Do We Need to Know This?

Some values genuinely shouldn't change during a program's execution — a fixed conversion rate, a fixed limit, and so on. Marking them const lets the compiler catch accidental modification for you, instead of relying on the programmer to remember not to change it.

Real-Life Example

Like a sealed, tamper-proof container versus a regular reusable one. A regular variable can be refilled or changed anytime, but a const variable is sealed after its initial value is set — there's no reopening it.

Visual Explanation

Regular variable 
int a = 5;

// later in the program...
a = 10;   // allowed — a can change
Constant 
const int a = 5;

// later in the program...
a = 10;   // compiler error — a is const

A regular variable can be reassigned freely. A const variable is given its value once, at declaration, and any later attempt to change it is rejected by the compiler.

Syntax

  • const type name = value; — for example, const int MAX = 100;.
  • The const keyword goes before the type.
  • A constant must be given an initial value at declaration, since it can never be assigned afterward.

Step-by-Step Code

Here is add.c again, this time marking its two fixed inputs as const:

add.c 
#include <stdio.h>

int main() {
    const int a = 5;
    const int b = 3;
    int sum = a + b;
    printf("The sum of %d and %d is %d\n", a, b, sum);
    return 0;
}

a and b are now marked const, since their values are fixed inputs to this program and never need to change. sum stays a regular (non-const) variable, since it's a computed result. Note that this same output — The sum of 5 and 3 is 8 — is unchanged; only the guarantee that a and b can never change has been added.

Common Mistakes

  • Trying to declare a const without an initial value — const int a; is invalid, since a constant can never be assigned afterward and so must be given a value at declaration.
  • Trying to reassign a const variable later in the code — this causes a compiler error, by design.

Debugging Tips

If the compiler reports an error like "assignment of read-only variable," check whether you're trying to modify something declared const.

Best Practices

  • Use const for any value you know shouldn't change once set — it documents your intent clearly and lets the compiler help enforce it, rather than relying on memory or comments alone.

FAQs

Q: Does using const make a program run faster?
A: Not typically a meaningful speed difference at this level — its main benefit is compiler-enforced safety and clearer intent, not performance.

Q: Can function parameters be const too?
A: Yes, though that's beyond this course's current scope.

Key Takeaways

  • const marks a variable as unchangeable after its initial declaration, letting the compiler enforce that a value stays fixed.
  • A constant must be given an initial value at declaration, since it can never be assigned afterward.
  • In this page's example, the fixed inputs a and b are marked const, while the computed result sum stays a regular variable, since it's calculated rather than fixed.
