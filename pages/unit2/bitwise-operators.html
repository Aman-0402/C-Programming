Bitwise Operators

Operators that work directly on the individual binary bits making up a number — &, |, ^, ~, <<, and >> — extending add.c one more time.

What Is It?

Bitwise operators work directly on the individual binary bits (0s and 1s) that make up a number's actual memory representation. C provides six of them: & (AND), | (OR), ^ (XOR), ~ (NOT/complement), << (left shift), and >> (right shift). These are different from the logical operators (&&, ||, !) covered earlier, which work on whole true/false values, not individual bits.

Why Do We Need to Know This?

Bitwise operators connect directly back to how numbers are actually stored in memory — a concept touched on since Unit 1's discussion of binary and machine code. They're commonly used in low-level, performance-critical, or hardware-adjacent C code, where working with individual bits matters.

Real-Life Example

Think of a panel of individual light switches — each bit is its own independent switch (on/off), and bitwise operators let you check, combine, or flip specific switches without affecting the others.

Syntax

syntax 
result = operand1 & operand2;   // bitwise AND
result = operand1 | operand2;   // bitwise OR
result = operand1 ^ operand2;   // bitwise XOR
result = ~operand;              // bitwise NOT (unary, only one operand)
result = operand << numBits;    // left shift by numBits positions
result = operand >> numBits;    // right shift by numBits positions

Note that ~ is a unary operator — it takes only one operand, unlike the other five, which are binary (take two operands).

Step-by-Step Code

Extending add.c with a bitwise AND operation:

add.c 
#include <stdio.h>

int main() {
    int a = 5;
    int b = 3;
    int bitwiseAnd = a & b;

    printf("Bitwise AND: %d\n", bitwiseAnd);

    return 0;
}
  • 5 in binary is 0101, and 3 in binary is 0011.
  • & compares each bit position: a bit in the result is 1 only if both corresponding bits are 1.
  • Comparing 0101 and 0011 position by position — rightmost bits: 1 & 1 = 1; next: 0 & 1 = 0; next: 1 & 0 = 0; leftmost: 0 & 0 = 0 — giving 0001, which is 1 in decimal.

Dry Run

Tracing through execution with a = 5 (0101), b = 3 (0011):

  • Bit position 0 (rightmost): 1 & 1 = 1
  • Bit position 1: 0 & 1 = 0
  • Bit position 2: 1 & 0 = 0
  • Bit position 3 (leftmost): 0 & 0 = 0
  • Result: 0001, which is 1 in decimal, stored in bitwiseAnd.

The exact output is:

output 
Bitwise AND: 1

Common Mistakes

  • Confusing bitwise & / | with logical && / || — bitwise operators work on individual bits and can produce any resulting number, while logical operators work on whole true/false values and only ever produce 1 or 0. This is one of the most common C mixups.
  • Forgetting that shift operators (<<, >>) move bit positions, which has the side effect of multiplying/dividing by powers of 2 (e.g. a << 1 doubles a's value) — a real but easy-to-forget connection.

Debugging Tips

If a calculation using & or | gives a surprising numeric result (not just 0 or 1), remember these are bitwise operators working on individual bits, not logical operators working on true/false — trace the binary representation by hand if unsure.

Best Practices

  • Use bitwise operators deliberately for genuinely bit-level tasks, like flags or hardware-adjacent code.
  • Don't accidentally reach for & / | when you actually meant the logical && / || from the Logical Operators page — double-check which one your code actually needs.

FAQs

Q: Are bitwise operators commonly used in typical beginner programs?
A: Not usually for everyday tasks, but they're important for low-level, embedded, or performance-critical code, and worth recognizing when you see them.

Q: Is ~ (bitwise NOT) the same as ! (logical NOT)?
A: No — ~ flips every individual bit of a number; ! reverses a whole true/false value; they behave very differently.

Key Takeaways

  • C provides 6 bitwise operators — &, |, ^, ~, <<, >> — that work on the individual bits of a number's memory representation.
  • Bitwise operators (&, |) are easily confused with logical operators (&&, ||) — bitwise ones work bit-by-bit and can produce any value, while logical ones only ever produce 1 or 0.
  • Shift operators (<<, >>) move bit positions, with the side effect of multiplying or dividing by powers of 2.
