Values

The last stop of this chapter — the actual data that variables and constants hold, and why "variable" and "value" are not the same thing.

1. What Is It?

A value is the actual data stored at a given moment — the concrete number, character, or other piece of data that a variable or constant currently holds. In int a = 5;, a is the variable (the named storage location), and 5 is the value currently stored inside it.

2. Why Do We Need to Know This?

Distinguishing "variable" (the named container) from "value" (the actual data inside it) makes it much clearer exactly what changes and what stays fixed as a program runs. It also sets up more precise vocabulary you'll need later in this course, for concepts like l-value and r-value.

3. Real-Life Example

Think of a labeled storage box named a — that label is the variable. Whatever happens to be inside the box right now, say 5, is the value. The box's label stays the same even as its contents change over time.

4. Visual Explanation

Here is the final memory state of this chapter's add.c example, with each cell explicitly labeled by its variable name (the label) and the value (the number currently inside it):

variable name: a
value: 5
variable name: b
value: 3
variable name: sum

a, b, and sum are the variables — the named boxes. 5, 3, and 8 are their values — what's actually sitting inside each box right now.

5. Dry Run

add.c 
#include <stdio.h>

int main() {
    const int a = 5;
    const int b = 3;
    int sum = a + b;
    printf("The sum of %d and %d is %d\n", a, b, sum);
    return 0;
}

Tracing through this one more time, in terms of values:

  • The value 5 is stored in a.
  • The value 3 is stored in b.
  • The value of a + b, which is 8, is stored in sum.
  • The values of a, b, and sum — 5, 3, and 8 — are substituted into the printf output, producing "The sum of 5 and 3 is 8".

Every variable and constant in this program ultimately just holds a value — that's the idea this whole chapter has been building toward.

6. Common Mistakes

  • Using "variable" and "value" interchangeably, as if they mean the same thing. They don't — one is the named container, the other is what's currently inside it.
  • Assuming a value can never change just because it happens to be correct right now. A regular (non-const) variable's value absolutely can change later in the program — only a const locks it in place.

7. Debugging Tips

When debugging by printing variables, remember you're printing their current values at that specific moment in execution — the very same variable printed earlier or later in the program could hold a completely different value.

8. Best Practices

  • Be precise with vocabulary when discussing code with others: saying "the value of a" versus "the variable a" communicates something specific and different.
  • Using these terms carefully avoids confusion, especially when working with a team or explaining a bug.

13. Mini Project

Declare Your Own Variables and Constants.

Pick a small program idea of your own — for example, calculating the area of a rectangle, converting a temperature, or tracking a simple score. Then, following everything you've learned in this chapter, write out appropriate variable and constant declarations for it:

  • Correct identifier names, following the naming rules from earlier in this chapter.
  • Sensible use of const wherever a value shouldn't change once set.
  • A brief note next to each declaration describing what value it holds.
rectangle.c (example sketch) 
#include <stdio.h>

int main() {
    const float pi = 3.14f;   // fixed constant, never changes
    int length = 10;          // variable, holds the rectangle's length
    int width = 4;            // variable, holds the rectangle's width
    int area = length * width; // variable, holds the computed area

    printf("Area = %d\n", area);
    return 0;
}

14. FAQs

Q: Is "value" a C-specific term, or a general programming concept?
A: It's general — every programming language has this same variable/value distinction, just with different specific syntax.

Q: Will there be more precise value-related terms later in this course?
A: Yes — concepts like l-value and r-value refine this idea further, and are covered in a later Unit 2 sub-project.

15. Key Takeaways

  • A variable is the named container; a value is the actual data currently stored inside it — the two are related but not the same thing.
  • A regular variable's value can change over the course of a program's execution, while a const's value is locked in at declaration and can never change afterward.
