Type Conversion

C changes a value from one data type to another in two ways — one automatic, one deliberate — and knowing the difference prevents surprising, silent data loss.

1. What Is It?

Type conversion is changing a value from one data type to another. C does this in two ways: implicit conversion, where the compiler converts a value automatically — sometimes losing data in the process, as previewed on the Integer Types page when a float got truncated by assigning it to an int — and explicit conversion, better known as casting, where the programmer deliberately requests a conversion using (type) syntax.

2. Why Do We Need to Know This?

Understanding both kinds of conversion prevents two common problems: accidentally losing data through unintended implicit conversion, and not knowing exactly how to force a conversion on purpose when you actually need one. Casting is one of the most useful tools for writing arithmetic that behaves the way you actually expect.

3. Real-Life Example

Think of converting currency. Sometimes a machine automatically rounds to the nearest whole unit without asking — that's implicit conversion, and it can quietly lose the fractional amount. Other times, you deliberately request an exact, specific conversion rate on purpose — that's explicit conversion, a choice you make intentionally.

4. Visual Explanation

Here are both kinds of conversion side by side, using the same value:

implicit vs. explicit 
int a = 5.9;          // implicit — silently truncates to 5
int b = (int)5.9;     // explicit — the (int) cast makes the truncation obvious

Both lines end up storing 5. The first does it silently, as a side effect of assigning a float value to an int variable. The second does the exact same truncation, but the (int) cast makes it an obvious, deliberate choice to anyone reading the code.

5. Syntax

syntax 
(targetType)expression

// examples:
(int)height
(float)age

The cast — the target type in parentheses — goes immediately before the value or expression being converted. It doesn't change the original variable at all; it only changes how that one expression's value is treated at that moment.

6. Step-by-Step Code

Using explicit casting with variables in the same spirit as add.c from earlier in this course:

divide.c 
#include <stdio.h>

int main() {
    int a = 5;
    int b = 3;
    float result = (float)a / b;

    printf("Result: %.2f\n", result);

    return 0;
}
  • int a = 5; and int b = 3; declare two whole-number variables.
  • Without the (float) cast, a / b would perform integer division — 5 / 3 would evaluate to 1, truncated, losing the decimal part entirely.
  • (float)a casts a to a floating-point value before the division happens, which forces the whole expression (float)a / b to be evaluated as floating-point division.
  • The result is the more precise 1.67, stored in result and printed with %.2f to show two decimal places.

This connects directly to a common real beginner mistake: expecting division between two int variables to automatically give a decimal answer, when C actually performs integer division unless you explicitly cast at least one operand first.

7. Common Mistakes

  • Expecting a / b, where both a and b are int, to give a decimal result without any casting — it won't; integer division truncates the result to a whole number.
  • Forgetting how many operands actually need casting — a single cast on just one operand, as shown above, is enough to force the whole expression into floating-point division, but it's easy to forget which one needs it (or to mistakenly think both do).

8. Debugging Tips

If a division result looks unexpectedly like a whole number when you expected a decimal, check whether both operands are int — if so, you likely need an explicit (float) or (double) cast on at least one of them before the division happens.

9. Best Practices

  • Be explicit about conversions you intend, using casts, rather than relying on implicit conversion — this makes your intent clear to future readers (including yourself) and avoids silent, surprising data loss.

14. FAQs

Q: Is casting the same as changing a variable's actual declared type?
A: No — a cast only affects how one specific expression's value is treated in that moment; the variable itself keeps its originally declared type.

Q: Can you cast in the other direction, e.g. float to int?
A: Yes — (int)someFloatValue truncates the decimal part, the same underlying truncation behavior covered on the Integer Types page.

15. Key Takeaways

  • C converts values two ways: implicit conversion (automatic, sometimes lossy) and explicit conversion/casting (deliberate, using (type) syntax).
  • The casting syntax is (targetType)expression, placed immediately before the value being converted — it doesn't change the original variable's declared type.
  • The most important practical case to recognize: dividing two int variables performs integer division and truncates the result, unless at least one operand is cast to float or double first.
