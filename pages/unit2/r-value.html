R-value

The other half of the assignment vocabulary — an expression that produces a value to be read, without necessarily having its own storage location.

What Is It?

An r-value ("read value") is an expression that produces a value but doesn't necessarily refer to an addressable storage location — something that can legally appear on the right side of an assignment. In a = 5;, the literal 5 is an r-value. In sum = a + b;, the entire expression a + b is also an r-value — it's a computed result with no storage location of its own, just a value waiting to be read and stored somewhere.

Why Do We Need to Know This?

Contrasting r-values with the l-values covered on the previous page completes the vocabulary needed to precisely describe why some expressions can be assignment targets (l-values) while others can only ever be assigned from (r-values). This distinction is exactly what a compiler is checking when it accepts or rejects an assignment.

Real-Life Example

Think of the actual letter placed inside a mailbox. The mailbox itself is the l-value — a fixed, addressable location. The letter's content is the r-value — what actually gets delivered and read. The letter's content doesn't need its own separate mailbox to be valid; it just needs to be read and, if desired, placed into a mailbox.

Visual Explanation

Here's sum = a + b; from the add.c example, broken down into its l-value and r-value halves:

l-value / r-value breakdown 
sum   =   a + b;
 |          |
 |          +-- r-value: a computed result being read
 |              (a and b are read as r-values here)
 |
 +-- l-value: the addressable target the result is stored into

The right side, a + b, is computed first — that computation is the r-value. Its resulting value is then stored into sum, the l-value.

Common Mistakes

  • Assuming an r-value can also appear on the left of an assignment — it generally can't. A computed expression like a + b has no storage location to assign into, so it can only ever be read from, not written to.
  • Confusing "r-value" with "a fixed literal only." Variables themselves can also act as r-values when their current value is being read — for example, a in sum = a + b; is being read as an r-value there, even though a is also an l-value in other contexts, such as when it's assigned to directly.

Debugging Tips

If you accidentally write something like (a + b) = sum;, expect a compiler error. a + b is a pure r-value with no storage location to assign into, so the compiler will reject it — typically with an "lvalue required" style error message.

Best Practices

  • Think of every assignment as "l-value = r-value" — the right side is read/computed first, and only then is its resulting value stored into the left side's addressable location.

FAQs

Q: Is r-value just another name for "the answer"?
A: Loosely, yes — it's the readable/producible value side of an expression, though the formal term is more precise than that casual description.

Q: Do I need to memorize l-value/r-value terminology for everyday coding?
A: Not strictly for writing basic programs, but understanding the concept helps make sense of certain compiler errors and more advanced topics later.

Key Takeaways

  • An r-value is an expression that produces a value to be read — usable on the right side of an assignment — without necessarily having its own storage location.
  • Every assignment pairs an l-value with an r-value: l-value = r-value; — the right side is read/computed, then stored into the left side's location.
  • The same variable can act as either an l-value or an r-value depending on context — assigned to in one place, read from in another.
