Integer Types

A new chapter, a new running example — meet int, C's basic whole-number data type, and start building a small "student profile" program that will grow over the next few pages.

What Is It?

int is C's basic integer data type — it stores whole numbers (numbers with no decimal point), like 20, -5, or 0. C also provides size variants — short, long, and long long — for controlling how much memory an integer uses and what range of values it can hold, though plain int is by far the most common choice for everyday use.

Why Do We Need to Know This?

Choosing the right data type for a piece of data — starting with recognizing when a whole number is appropriate — is foundational to writing correct, memory-efficient C programs. Every value you store needs a type, and int is usually the very first one you'll reach for.

Real-Life Example

Think of choosing a container size for counting whole items, like eggs in a carton — you wouldn't use a container designed for liquid measurements (which can hold fractional amounts) to count discrete whole things. An int is exactly that kind of "whole item" container: it's built for counting, not for measuring precise fractional amounts.

Visual Explanation

Here's how a single int variable holds its whole-number value in memory:

int age
20

age holds the whole number 20 — no decimal point, nothing fractional. That's the defining trait of an int.

Syntax

syntax 
int variableName = wholeNumberValue;

// example:
int age = 20;

C also offers size variants of int — short int, long int, and long long int — for situations where you need a smaller or much larger range of whole numbers. At this stage, it's enough to know these variants exist; their specific byte sizes are covered later.

Step-by-Step Code

This page starts a small recurring example — a student profile program — beginning with just its whole-number part:

profile.c 
#include <stdio.h>

int main() {
    int age = 20;

    printf("Age: %d\n", age);

    return 0;
}
  • int age = 20; declares an int variable named age and stores the whole number 20 in it.
  • printf("Age: %d\n", age); prints that value, using %d — the format specifier for a whole number (int).

This is just the beginning: over the next few pages, this same profile.c program will grow to include a height (floating point), a grade (character), and a name (string) — each new data type building on this same running example.

Common Mistakes

  • Trying to store a decimal value in an int — it gets silently truncated. Storing 5.9 in an int would just become 5, losing the decimal part entirely (covered more precisely on the next page).
  • Assuming all whole numbers need the largest integer type available. Plain int is sufficient for the vast majority of everyday values — reaching for long long "just in case" usually isn't necessary.

Debugging Tips

If a calculation you expected to have a decimal result instead shows a whole number with no decimal part, check whether you're accidentally storing it in an int instead of a floating-point type — that silent truncation is a very common source of this kind of bug.

Best Practices

  • Use plain int for everyday whole-number values unless you have a specific reason — like memory constraints or a huge range requirement — to reach for a size variant.
  • Only introduce short, long, or long long when your expected values genuinely call for it, rather than by default.

FAQs

Q: Is int always the same size on every system?
A: Not guaranteed by the language itself — it can vary by platform/compiler, though it's commonly 4 bytes on modern systems.

Q: Should I always use long instead of int to be safe?
A: No — plain int suffices for the vast majority of everyday values; oversizing everything isn't necessary or considered good practice.

Key Takeaways

  • int stores whole numbers — no decimal point — and is C's most commonly used numeric type.
  • Storing a decimal value in an int silently truncates it, which is a real and common source of bugs.
