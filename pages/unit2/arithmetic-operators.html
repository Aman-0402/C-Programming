Arithmetic Operators

The five operators behind almost every calculation a C program performs — extending add.c with subtraction, multiplication, and the modulus (remainder) operator.

1. What Is It?

Arithmetic operators perform mathematical calculations on numeric values: + (addition, already seen in add.c), - (subtraction), * (multiplication), / (division), and % (modulus — the remainder left over after integer division).

2. Why Do We Need to Know This?

These are the most commonly used operators in any program — nearly every calculation a program performs relies on some combination of them.

3. Real-Life Example

Think of the basic operations on a calculator — add, subtract, multiply, divide — plus one extra C provides that a typical calculator doesn't emphasize: modulus, which tells you the "leftover" amount after dividing evenly as many times as possible. For example, splitting 7 candies among 2 people gives 3 each with 1 left over — that leftover 1 is the modulus result.

4. Syntax

syntax 
result = operand1 operator operand2;

// examples:
int diff = a - b;
int product = a * b;
int remainder = a % b;

% only works with integer operands in C.

5. Step-by-Step Code

Extending add.c with the three new operators:

add.c 
#include <stdio.h>

int main() {
    int a = 5;
    int b = 3;
    int sum = a + b;
    int diff = a - b;
    int product = a * b;
    int remainder = a % b;

    printf("Sum: %d\n", sum);
    printf("Difference: %d\n", diff);
    printf("Product: %d\n", product);
    printf("Remainder: %d\n", remainder);

    return 0;
}
  • int diff = a - b; subtracts b from a: 5 - 3 = 2.
  • int product = a * b; multiplies a and b: 5 * 3 = 15.
  • int remainder = a % b; divides a by b and keeps only the leftover: 5 % 3 = 2, the amount left over after dividing 5 by 3 once.

6. Dry Run

Tracing through execution with a = 5 and b = 3:

  • sum is set to a + b = 5 + 3 = 8.
  • diff is set to a - b = 5 - 3 = 2.
  • product is set to a * b = 5 * 3 = 15.
  • remainder is set to a % b = 5 % 3 = 2.

The exact output is:

output 
Sum: 8
Difference: 2
Product: 15
Remainder: 2

7. Common Mistakes

  • Confusing % (modulus/remainder) with percentage — it has nothing to do with percentages in C, it's strictly the remainder of integer division.
  • Forgetting that / between two ints performs integer division, truncating any decimal part of the result — a callback to the Type Conversion page's coverage of this exact issue.

8. Debugging Tips

If a % (modulus) result looks unexpected, remember it's specifically "what's left over after dividing as many whole times as possible" — trace the division by hand if the result is surprising.

9. Best Practices

  • Use parentheses to make the order of operations in a complex arithmetic expression explicit and readable, even when not strictly required by operator precedence rules (a topic covered more fully in a later chapter).

14. FAQs

Q: Can I use % with float or double values in standard C?
A: Not with the plain % operator — it's specifically for integers.

Q: Does the order of arithmetic operators matter in a single expression?
A: Yes — this is called operator precedence, covered in a later chapter; for now, using parentheses to be explicit is a safe habit.

15. Key Takeaways

  • C provides 5 arithmetic operators: +, -, *, /, and %, covering addition, subtraction, multiplication, division, and modulus (remainder).
  • % gives the remainder left over after integer division — it has nothing to do with percentages.
  • / between two ints performs integer division, truncating any decimal part — a risk to remember from the Type Conversion page.
