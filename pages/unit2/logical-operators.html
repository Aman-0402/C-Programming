Logical Operators

The last topic in this chapter — how &&, ||, and ! combine multiple true/false conditions into a single overall decision, extending add.c one final time.

What Is It?

Logical operators combine multiple true/false conditions into a single overall true/false result. C provides three of them: && (AND — true only if both conditions are true), || (OR — true if at least one condition is true), and ! (NOT — reverses a condition's true/false result).

Why Do We Need to Know This?

Real decisions in programs usually depend on more than one condition at once — for example, "is this number positive and even?" Logical operators are how multiple relational checks, covered on the previous page, get combined into one overall decision that a program can act on.

Real-Life Example

Think of a two-key security system that requires both keys to be turned at once to open — that's && (AND) in action. Compare that to a system that accepts either of two valid keycards — that's || (OR). Logical operators express exactly these kinds of combined requirements.

Syntax

syntax 
condition1 && condition2   // AND — true only if BOTH are true
condition1 || condition2   // OR — true if AT LEAST ONE is true
!condition                 // NOT — reverses a condition's result

Just like relational results from the previous page, the overall result of a logical expression is an int: 1 for true, 0 for false.

Step-by-Step Code

Extending add.c with a combined relational + logical check:

add.c 
#include <stdio.h>

int main() {
    int a = 5;
    int b = 3;
    int bothPositive = (a > 0) && (b > 0);

    printf("Are both a and b positive? %d\n", bothPositive);

    return 0;
}
  • (a > 0) evaluates to true, giving 1.
  • (b > 0) also evaluates to true, giving 1.
  • Combined with &&, since both individual conditions are true, bothPositive becomes 1.

Dry Run

Tracing through execution with a = 5, b = 3:

  • (a > 0) is evaluated first: 5 > 0 is true.
  • (b > 0) is evaluated next: 3 > 0 is true.
  • && requires both to be true — since they are, bothPositive is set to 1.

The exact output is:

output 
Are both a and b positive? 1

If b were -3 instead, (b > 0) would become false. Even though (a > 0) is still true, && requires both sides to be true, so the overall result would become 0.

Common Mistakes

  • Confusing && (AND — requires both conditions) with || (OR — requires at least one) and using the wrong one for the intended logic.
  • Forgetting that ! reverses a result — !(a > b) means "NOT (a is greater than b)," which is the same as "a is less than or equal to b."

Debugging Tips

If a combined condition isn't behaving as expected, break it into its individual pieces and check each relational comparison separately first, before combining them with && or ||. Isolating each piece makes it much easier to spot which part of the logic is wrong.

Best Practices

  • Use parentheses generously around individual conditions inside a logical expression, even when not strictly required, to make the intended grouping unambiguous to any reader — for example, (a > 0) && (b > 0) rather than a > 0 && b > 0.

FAQs

Q: Is && the same as "and" written out as a word?
A: Conceptually yes, but C requires the specific symbols && / || / ! — it doesn't accept English words as operators.

Q: Can I combine more than two conditions with && or ||?
A: Yes — you can chain multiple conditions together, e.g. (a > 0) && (b > 0) && (a != b), though this course's examples keep it to two for clarity.
