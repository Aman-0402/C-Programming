Assignment Operators

Beyond the basic = assignment operator you've used throughout this course, C provides compound assignment operators — +=, -=, *=, /=, %= — that combine an operation with assignment in one concise step.

What Is It?

You've already been using the basic assignment operator (=) throughout this course to store values in variables. C also provides compound assignment operators that combine an operation with assignment in a single step: +=, -=, *=, /=, and %=. For example, total += a; is shorthand for total = total + a;.

Why Do We Need to Know This?

Compound assignment operators are extremely common in real-world C code — especially for accumulating totals or counters. Recognizing this shorthand is essential for reading typical C programs, since experienced programmers use it by default rather than writing out the longhand form every time.

Real-Life Example

Think of saying "add $5 to my account" instead of "my account balance is now my old balance plus $5." Both mean exactly the same thing, but the first is shorter and gets straight to the point — that's exactly what the compound assignment form does compared to writing out the full longhand assignment.

Syntax

syntax 
variable += value;   // shorthand for: variable = variable + value;
variable -= value;   // shorthand for: variable = variable - value;
variable *= value;   // shorthand for: variable = variable * value;
variable /= value;   // shorthand for: variable = variable / value;
variable %= value;   // shorthand for: variable = variable % value;

Each compound assignment operator takes the variable's current value, applies the operation with the right-hand value, and stores the result back into that same variable.

Step-by-Step Code

Extending add.c with compound assignment operators:

add.c 
#include <stdio.h>

int main() {
    int a = 5;
    int b = 3;
    int total = 0;

    total += a;
    total += b;

    printf("Total: %d\n", total);

    return 0;
}
  • total starts at 0.
  • total += a; is shorthand for total = total + a;, making total equal to 5.
  • total += b; is shorthand for total = total + b;, making total equal to 8 (5 + 3).

Dry Run

Tracing through execution with a = 5, b = 3:

  • total is initialized to 0.
  • total += a; runs: total = 0 + 5, so total becomes 5.
  • total += b; runs: total = 5 + 3, so total becomes 8.

The exact output is:

output 
Total: 8

Common Mistakes

  • Forgetting that compound assignment operators still perform the full operation before assigning — for example, total -= a; fully subtracts a from total, not the reverse.
  • Confusing += with =+. These are not the same: =+ is actually just a plain = followed by the unary + operator (total = (+a);) — a legal but completely different and confusing thing to accidentally write. It's a classic old-school C gotcha worth remembering.

Debugging Tips

If a running total seems to only be growing by the last value added instead of accumulating across every step, check that you're actually using += (which accumulates onto the existing value) rather than accidentally resetting it with a plain = each time.

Best Practices

  • Use compound assignment operators whenever you're accumulating or updating a variable based on its own previous value — it's more concise, and it immediately signals "update in place" to anyone reading your code.

FAQs

Q: Are compound assignment operators required, or just convenient shorthand?
A: Just convenient shorthand — total = total + a; works identically, but += is more concise and idiomatic.

Q: Does every arithmetic operator have a compound assignment version?
A: The 5 covered here — +=, -=, *=, /=, %= — cover the arithmetic operators from the previous chapter. Other compound assignment operators exist for bitwise operators too, covered later in this group.

Key Takeaways

  • C provides 5 compound assignment operators: +=, -=, *=, /=, and %=.
  • Each one is shorthand for the longhand form — total += a; means exactly total = total + a;.
  • Compound assignment operators are especially common for accumulator-style patterns, where a variable is updated based on its own previous value.
