Variables

A variable is a named storage location in memory that holds a value which can change while the program runs — this is the page where add.c stops hardcoding its answer and actually starts computing it.

1. What Is It?

A variable is a named storage location in memory that holds a value which can change while the program runs. Declaring int a = 5; creates a variable named a, of type int, initially holding the value 5.

2. Why Do We Need to Know This?

Variables are what make programs actually do things with changing data, rather than just printing fixed, hardcoded text. This is the single biggest step up from everything shown in Unit 1 — from here on, your programs can compute, not just recite.

3. Real-Life Example

Like a labeled storage box that can hold different contents over time — the box's label (a) stays the same, but what's inside it (the value) can be swapped out.

4. Visual Explanation

After this page's example program runs, memory holds three labeled variables like this:

a
5
b
3
sum
8

Each box's label (its variable name) never changes, but the value it holds can — that's exactly what "variable" means.

5. Syntax

Declaring a variable follows this pattern:

syntax 
type name = initialValue;

For example, int a = 5;. The type (like int) tells the compiler what kind of data it holds and how much memory to reserve; the name is the identifier; = initialValue is optional at declaration time — a variable can be declared without a value, e.g. int a;, and assigned later.

6. Step-by-Step Code

Here is add.c, evolved from Unit 1's hardcoded version to one that uses real variables:

add.c 
#include <stdio.h>

int main() {
    int a = 5;
    int b = 3;
    int sum = a + b;
    printf("The sum of %d and %d is %d\n", a, b, sum);
    return 0;
}

int a = 5; declares a variable named a holding the value 5. int b = 3; declares a variable named b holding the value 3. int sum = a + b; declares a variable named sum, whose value is computed by adding a and b together. The printf line now uses %d placeholders to print the actual variable values instead of a hardcoded string — each %d prints an integer value, in order, from the arguments that follow (full format-specifier detail is covered later in this course).

8. Common Mistakes

  • Using a variable before declaring it — the compiler won't recognize the identifier yet.
  • Forgetting to give a variable an initial value and then trying to use its value before ever assigning one — its value is unpredictable/undefined until explicitly set.
  • Mismatching the type, e.g. trying to fit a value that doesn't fit an int (covered more deeply in a later topic).

9. Debugging Tips

If a variable's printed value looks wrong or garbage-like, check whether it was actually assigned a value before being used — an uninitialized variable's value is unpredictable, not automatically zero.

10. Best Practices

  • Always initialize variables with a sensible starting value when you declare them, unless you have a specific reason not to.
  • Choose clear, descriptive variable names, following the Identifiers and Naming Rules guidelines from earlier in this unit.

15. FAQs

Q: Can a variable's type change after it's declared?
A: No, not in C — a variable's type is fixed once declared.

Q: Do I have to initialize a variable when I declare it?
A: No, it's optional syntactically, but strongly recommended as best practice.

16. Key Takeaways

  • A variable is a named storage location in memory that holds a value which can change while the program runs.
  • A variable must be declared before it is used, and should generally be initialized at the same time.
  • add.c now computes its result from real variables instead of hardcoding the answer as a fixed string.
